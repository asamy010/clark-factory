## ✅ ما تم في هذه الجلسة

### 1. البنية التحتية (Data Structures)

#### حقول جديدة لـ `fabrics` و `accessories`:
- `stock`: الرصيد الحالي (رقم)
- `minStock`: الحد الأدنى للتنبيه
- `avgCost`: متوسط التكلفة (weighted avg)
- `lastReceiveDate`: تاريخ آخر استلام
- `qtyPerPiece` (للـ accessories فقط): كمية الإكسسوار لكل قطعة تامة

#### Collections جديدة في `configDoc`:
- `stockMovements[]`: سجل كامل لحركات المخزن (دخول/خروج/تسوية)
- `purchaseReceipts[]`: فواتير الاستلام
- `purchaseOrders[]`: أوامر الشراء (اختيارية)

#### Settings جديدة:
```javascript
purchaseSettings: {
  stockEnabled: false,           // Toggle التفعيل
  stockActivationDate: "",       // تاريخ التفعيل
  blockOnInsufficientStock: true, // Hard block عند القص
  autoDeductOnCut: true,          // خصم تلقائي
  receiptPrefix: "REC-",
  poPrefix: "PO-"
}
```

### 4. UI الكاملة للصفحة

#### Header ديناميكي:
- يعرض حالة المخزن (مفعّل / غير مفعّل)
- تاريخ التفعيل
- أزرار: "🚀 تفعيل المخزن" (لو غير مفعّل) أو "➕ رصيد ابتدائي" + "إيقاف" (لو مفعّل)

#### Sub-tabs (4 أقسام):
1. **📦 المخزن** — ✅ جاهز
2. **📥 الاستلامات** — 🔒 قادم (جلسة 2)
3. **📋 أوامر شراء** — 🔒 قادم (جلسة 5)
4. **👥 كشوف الموردين** — 🔒 قادم (جلسة 4)

#### صفحة المخزن الكاملة:

**4 إحصائيات أعلى الصفحة:**
- قيمة المخزن (خامات)
- قيمة المخزن (إكسسوار)
- أصناف ناقصة (أقل من minStock)
- أصناف نفذت (stock = 0)

**تبديل بين أنواع:**
- 🧵 الخامات
- 🪡 الإكسسوار

**فلاتر متقدمة:**
- بحث (debounced 200ms)
- ترتيب: الاسم / الرصيد / القيمة / الناقص أولاً
- إخفاء الأصناف الصفرية

**جدول شامل للمخزن:**
- الاسم
- الرصيد (ملون حسب الحالة)
- الوحدة
- qty/قطعة (للإكسسوار فقط — قابل للتعديل inline)
- الحد الأدنى (قابل للتعديل inline)
- متوسط التكلفة
- القيمة الإجمالية
- آخر استلام
- الحالة: متاح / ناقص / نافذ

**جدول حركات المخزن (آخر 50):**
- التاريخ، النوع (دخول/خروج/رصيد ابتدائي)، الصنف، الكمية، السعر، المرجع، بواسطة

### 5. Popups متكاملة

#### Popup تفعيل المخزن:
- شرح واضح لما سيحدث
- اختيار تاريخ التفعيل
- تأكيد قبل التفعيل

#### Popup الرصيد الابتدائي:
- تبديل بين خامات/إكسسوار
- جدول بكل العناصر
- إدخال: الكمية + تكلفة الوحدة
- حساب القيمة تلقائياً
- حفظ فقط العناصر اللي فيها بيانات

## 💡 ملاحظات مهمة

### المخزن "مُعطَّل" افتراضياً
تم تصميم النظام بحيث **لن يؤثر على المصنع** قبل التفعيل الصريح. الحقول الجديدة ستظهر كـ 0 فقط، ولن يتم خصم أي شيء من أي أوردر.

### الأوردرات الحالية آمنة
بما إن `stockActivationDate` يُستخدم للفلترة، الأوردرات المسجلة قبل التفعيل لن تُخصَم.

### الرصيد الابتدائي يُسجَّل كـ `stockMovement`
بنوع `"opening"` — يظهر في سجل الحركات كنقطة بداية واضحة.

### الاستلامات والحركات مش في الـ UI الحالي
هذه الجلسة: **view-only** للمخزن. جلسة 2 ستضيف إمكانية إضافة استلامات جديدة.
