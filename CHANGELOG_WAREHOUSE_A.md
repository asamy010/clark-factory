### 3. WarehousePg Component الكامل

#### 🎯 Sub-tab 1: نظرة عامة (Overview)
- **4 Interactive Stats Cards** (كل واحد clickable ليفتح الـ sub-tab المناسب):
  - 🧵 الخامات (قيمة + عدد + تحذيرات)
  - 🪡 الإكسسوار
  - 👕 الجاهز (عدد موديلات + قطع)
  - ➕ منتجات عامة
- **Big Card:** إجمالي قيمة المخازن الكلية
- **تنبيهات أصناف ناقصة:** جدول ملون لكل الأصناف تحت minStock
- **آخر 10 حركات:** preview سريع للنشاط

#### 🧵 Sub-tab 2: الخامات
- جدول شامل (نفس المحتوى الموجود في "مشتريات")
- فلاتر: بحث + ترتيب + إخفاء الصفرية
- Inline editing لـ minStock
- زر **⇅** (حركة يدوية) لكل خامة

#### 🪡 Sub-tab 3: الإكسسوار
- نفس جدول الخامات
- **Inline editing لـ qtyPerPiece** (كم قطعة في الموديل)
- زر حركة يدوية لكل إكسسوار

#### 👕 Sub-tab 4: الجاهز
- Splash card مع زر "📥 فتح تبويبة الجاهز"
- **Cross-tab navigation** عبر custom event `goto-tab`
- جدول بالموديلات التي لها رصيد جاهز:
  - عدد القطع المقصوصة
  - المسلم للعميل
  - الرصيد الجاهز (ملون)
  - الحالة
- يعتمد على `calcOrder` و `getConfirmedStock` الموجودين

#### ➕ Sub-tab 5: منتجات عامة (جديد كلياً)
- زر "➕ منتج جديد"
- فلاتر: بحث + فئة + ترتيب + إخفاء الصفرية
- جدول بعرض: اسم + فئة (badge) + رصيد + وحدة + حد أدنى + متوسط التكلفة + قيمة + حالة + إجراءات
- **3 إجراءات** لكل منتج:
  - ⇅ حركة جديدة
  - ✏️ تعديل
  - 🗑 حذف
- Empty state مع دعوة لإضافة أول منتج

#### 📊 Sub-tab 6: سجل الحركات الموحد (Movements)
**5 فلاتر قوية:**
- 🔍 بحث (اسم الصنف + ملاحظات) debounced 200ms
- نوع الصنف: كل / خامات / إكسسوار / منتجات
- نوع الحركة: كل / دخول / خروج / ابتدائي / تسوية
- تاريخ من
- تاريخ إلى

**الجدول:**
- التاريخ
- نوع الصنف (badge ملون: 🧵 خامة / 🪡 إكسسوار / ➕ منتج / 👕 جاهز)
- نوع الحركة (badge ملون: ↓ دخول / ↑ خروج / ◉ ابتدائي / ⟲ تسوية)
- اسم الصنف
- الكمية (مع إشارة ± ملونة)
- السعر
- المرجع (sourceType أو notes)
- بواسطة

**Sticky header** في الجدول + عدّاد النتائج.

### 4. 2 Popups Interactive

#### Popup إنشاء/تعديل منتج:
- اسم + فئة (dropdown) + وحدة
- سعر + حد أدنى
- ملاحظات (textarea)
- تنبيه: "أضف الرصيد الابتدائي من حركة جديدة ⇅"

#### Popup حركة يدوية:
- **Current stock** معروض فوق
- **3 أنواع حركة** بأزرار كبيرة ملونة:
  - ↓ **دخول** (أخضر) — يضيف للمخزن + weighted avg للتكلفة
  - ↑ **خروج** (أحمر) — يخصم مع شيك المتاح
  - ⟲ **تسوية** (أصفر) — تحدد الرصيد الجديد مباشرة
- الكمية + التاريخ
- السعر (يظهر فقط عند "دخول")
- ملاحظات

### 5. Logic ذكي

#### Weighted Average (في حركات الدخول):
```javascript
oldValue = oldStock × oldAvg
newValue = qty × price
totalStock = oldStock + qty
newAvgCost = (oldValue + newValue) / totalStock
```

#### Stock Check (في حركات الخروج):
- يشيك إن المطلوب <= المتاح
- لو أكبر: popup تحذير "المخزن غير كافي" مع التفاصيل

#### Adjust Mode:
- qty = الرصيد الجديد (مش delta)
- الحركة تُسجَّل بـ (qty - curStock) علشان الفرق يبقى واضح

### 6. Cross-Tab Navigation
- **Custom event `goto-tab`** للتنقل بين tabs من داخل component
- مثلاً: من المخازن → الجاهز → يفتح `stock` tab القديم
- استخدام: `window.dispatchEvent(new CustomEvent("goto-tab", {detail:"stock"}))`
