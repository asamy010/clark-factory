# 📱 Prompt احترافي — Mobile Professional Mode (Handheld-style)

> **النسخة الحالية:** CLARK V21.9.31
> **نوع المهمة:** Major UX feature — Mobile-first dedicated mode بـ entry screen + 3 modes متخصصة
> **علاقته بالـ prompts السابقة:** مستقل — لكن يستفيد من #10 (Tagging) للـ filtering في الـ collection mode

---

## 🔍 الفهم الصحيح من الكود

### ما هو موجود (Mobile responsiveness أساسية)

**1. Mobile detection في `App.jsx`:**
- السطر 648: `const w=useWin(); const isMob=w<768; const isTab=w>=768&&w<1100`
- الـ `isMob` بـ يـ passed كـ prop لكل page

**2. Existing mobile responsiveness:**
- كل صفحة بـ يـ check `isMob` ويـ adapt الـ layout
- لكن **الـ UI الأساسي بـ يفضل نفسه** — desktop-style بـ padding أصغر

**3. QR Infrastructure شغّال:**
- `src/utils/qr.js` — `loadQR`, `loadJsQR`, `scanQR`
- في CustDeliverPg السطور 1070-1074 — buttons لـ "بيع سريع" / "مرتجع سريع" / "تأكيد استلام"
- Pattern: window events للـ programmatic trigger:
  - `window.__qrSaleMode = "sale"` ثم `dispatchEvent("qr-sale-trigger")`
  - `window.__openPkg = pkgId` ثم `dispatchEvent("open-pkg")`

**4. Existing flows اللي ممكن نستفيد منهم:**
- `qrSale` flow (CustDeliverPg) — modal بـ steps
- `pendingRcv` flow — تأكيد استلام بـ QR scan
- `invAudit` flow — جرد بـ QR

### الـ Gaps الجوهرية

**Gap 1: مفيش Mobile-First Entry Screen**
- لما تفتح CLARK على موبايل: بـ يفتح نفس الـ home page الـ desktop
- مفيش "Welcome" screen يـ ask "إيه عاوز تعمل النهارده؟"

**Gap 2: مفيش Modes منفصلة:**
- المستخدم لازم يـ navigate عبر الـ tabs/menus (مش طبيعي على موبايل)
- مفيش "Warehouse Mode" / "Collection Mode" focused screens

**Gap 3: Smart QR Scanner مع Auto-Routing مفقود:**
- الـ QR scan الحالي **محصور في context محدد**
- مفيش standalone scanner يفتح من الـ home، يـ detect نوع الـ QR، ويـ route تلقائياً
- مفيش QR types موحّدة (customer QR? workshop QR?)

**Gap 4: Collection Mode مفقود:**
- "تحصيل دفعة" حالياً = navigate لـ CustDeliverPg → اختار عميل → scroll → فتح payment form → fill all fields
- مفيش flow مبسط: "list عملاء → tap → mini payment form → done"

**Gap 5: Touch UX optimization مش لـ handheld:**
- مفيش haptic feedback، voice commands، biometric

  QR Detection + Routing:
  
  function detectAndRoute(qrText){
    if(qrText.startsWith("MOD-")) → model_action
    if(qrText.startsWith("CUS-")) → customer_action
    if(qrText.startsWith("WS-"))  → workshop_action
    if(qrText.startsWith("EMP-")) → employee_action
    if(qrText.startsWith("INV-")) → invoice_action
    if(/^\+?20\d{10}$/) → customer phone lookup
    if(qrText.startsWith("http")) → portal link
    else → fallback to order lookup
  }
