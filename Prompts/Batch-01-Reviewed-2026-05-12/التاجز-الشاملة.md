### Tag Registry Schema المقترح (First-class objects)

```js
data.tagRegistry = [
  {
    id: "tag_vip",                          // stable ID (gid)
    name: "VIP",                            // display name (Arabic OK)
    nameLC: "vip",                          // lowercase for matching
    color: "#F59E0B",                       // hex
    icon: "⭐",                              // optional emoji
    description: "عميل مميز يستحق معاملة خاصة",
    appliesTo: ["customer", "supplier"],    // entity types
    createdBy, createdAt,
    usageCount: 0,                           // computed/cached
    archived: false,
    archivedAt?, archivedBy?,
    
    // Optional: linked auto-actions
    autoActions?: {
      onApply: [],   // future: trigger workflows
    }
  }
]
```

E. Filter helper

  src/utils/tags.js (جديد)
  
  export function filterByTags(entities, tagIds, mode = "OR") {
    if(!tagIds || tagIds.length === 0) return entities;
    return entities.filter(e => {
      const eTags = e.tags || [];
      if(mode === "AND"){
        return tagIds.every(t => eTags.includes(t));
      }
      return tagIds.some(t => eTags.includes(t));
    });
  }
  
  export function resolveTagsForDisplay(tagIds, tagRegistry) {
    if(!tagIds) return [];
    return tagIds.map(id => tagRegistry.find(t => t.id === id))
                 .filter(Boolean);
  }
  
  export function getTagUsageCount(tagId, data) {
    let count = 0;
    const entityArrays = [
      data.customers, data.suppliers, data.inventoryItems,
      data.orders, data.workshops, data.employees,
      data.salesInvoices, data.purchaseInvoices,
      data.treasury, data.checks
    ];
    entityArrays.forEach(arr => {
      if(Array.isArray(arr)){
        arr.forEach(e => {
          if(Array.isArray(e.tags) && e.tags.includes(tagId)) count++;
        });
      }
    });
    return count;
  }

## 💡 ملاحظات لـ Ahmed قبل ما تستخدم الـ برومبت

**ده فيتشر cross-cutting كبير (3-4 أسابيع لو هتعمل Phase 1 + 2)** لكنه بيغيّر الـ workflow بشكل جذري — زي Odoo بالظبط، tags بتظهر وتتفلتر في كل مكان.

**5 أسئلة الـ AI هيسألك عنها — جهز إجاباتك:**

1. **اللغة في الـ tag names:** Arabic فقط، English فقط، أم mixed؟ → توصيتي: mixed (الـ matching case-insensitive)

2. **مين يقدر يـ create tag جديد:** أي user عنده edit permission، ولا admin/manager فقط؟ → توصيتي: manager + admin (المحاسبين والمستخدمين العاديين بيختاروا من الموجود فقط — يمنع الـ fragmentation)

3. **Color palette:** preset (10-15 لون) ولا free hex picker؟ → توصيتي: preset — يحافظ على consistency الـ UI

4. **Shopify migration:** الـ tags الموجودة على customers تـ migrate تلقائياً لـ registry ولا admin يراجع أولاً؟ → توصيتي: auto-migrate في dry-run الأول (تطلع للأدمن preview)، بعد الـ approval يـ commit

5. **Phase 1 entities الأولوية:** أنهي entities الأولى؟ → توصيتي: customer (موجود فعلاً)، supplier، item، order — دول الـ 80% من الـ use cases

**أهم 3 قرارات معمارية:**

**1) Tags كـ first-class objects (registry):**
بدل ما الـ tags strings مجردة، كل tag بيكون له ID + color + description + appliesTo. ده اللي بيدي الـ "زي Odoo feeling" — لما تـ rename "VIP" في الـ settings، بـ يتغير everywhere instantly.

**2) IDs في الـ entities، مش names:**
`customer.tags = ["tag_vip", "tag_cairo"]` مش `["VIP", "Cairo"]`. ده يخلي الـ rename + archive + merge شغّالين بدون migration.

**3) Shopify adapter منفصل:**
الـ Shopify لسه بيـ accept strings. الـ sync بيـ resolve بين الـ IDs والـ names في الاتجاهين بدون ما يغير الـ Shopify API. الـ migration بيتعمل مرة واحدة فقط في Slice 4.

**نقطة technical حرجة — الـ Performance:**
لو عندك 5000 عميل و filter بـ 3 tags، الـ naïve approach = O(N×M) = 15000 lookups كل re-render. الـ AI لازم يـ use Set في filterByTags + memoization. لو مش هيعمل ده، الـ filtering هيـ lag بشكل واضح على iPad.
