## 🛡️ Architectural Decisions Documented

1. **Pushback per CLAUDE.md §0.1:** Ahmed طلب 8 توصيات في البداية، أنا اعترضت وحصلنا على 3 آمنين أولاً (182-184). بعدها كل feature جديدة كانت request منفصل.

2. **Per-customer discount precedence:** `delivery.discPct → customer.discount → 10`. الـdelivery override بـ يـ frozen at sale time (مش lookup live) عشان الـadmin يقدر يعدّل sess.custDisc بعد البيع بدون retroactive AR drift.

3. **Posted invoices never mutated:** الـdrafts تـ recompute عند الـupsert/merge، الـposted invoices تحتاج void + repost manual (Phase 3 deferred — يحتاج V21.9.90 COGS full flow أولاً).

4. **Zero migrations:** كل التغييرات في الـ11 versions additive. الـlegacy data بدون الـnew fields بـ يـ fall through الـprecedence chains.

5. **Build verification:** غير ممكن محلياً (no node/npm في الـshell). Vercel deploy هو الـbuild verifier الوحيد. لو الـbuild فشل في Vercel، الـdeploy ما هـ يتمش — production ما تتأثرش.
