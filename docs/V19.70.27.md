# V19.70.27 — Markazi Text across all PDFs

**Date:** 2026-05-06
**Type:** ux
**Status:** shipped

## What

User reviewed an HTML font preview comparing Mada / Markazi Text / Reem Kufi / Amiri / Cairo and chose **Markazi Text** for all PDFs in the application.

This version migrates every PDF and printable output to use Markazi Text as the primary font, with Cairo retained as a fallback. The on-screen UI (popups, toasts, login, dashboard) keeps Cairo unchanged.

## Two PDF paths, both updated

### 1. jsPDF (vector) PDFs — `src/utils/arabicPdf.js`
- Bulk delivery WhatsApp PDF (`buildDeliveryReceiptPdfBase64`)
- Available stock popup PDF (`buildAvailableStockPdfBase64`)

The bundled TTFs in `public/fonts/` were swapped:
```
- Amiri-Regular.ttf     (~430KB, calligraphic Naskh, removed)
- Amiri-Bold.ttf        (~410KB, removed)
+ MarkaziText-Regular.ttf  (~181KB, modern serif)
+ MarkaziText-Bold.ttf     (~181KB)
```

The font family is still aliased as `"Cairo"` in jsPDF's font registry — all `pdf.setFont("Cairo")` callers continue to work without modification. The actual rendered font is now Markazi Text.

Total bundled fonts: ~360KB (down from Amiri's ~840KB).

### 2. Browser-print outputs — multiple files
The CSS `font-family` was switched in every print/PDF output:

- `src/constants/index.js` — `PRINT_CSS` (the global stylesheet for all `printPage()` outputs)
- `src/utils/print.js` — main `printPage()` HTML + week-label envelope sheet
- `src/utils/printTemplates.js` — all 5 template CSS bodies
- `src/utils/htmlToPdf.js` — legacy CDN URL (mostly unused now)
- `src/pages/CustDeliverPg.jsx` — multiple inline CSS strings (group-print receipt, bulk receipt, stock report, delivery report, delivery log)
- `src/pages/TreasuryPg.jsx` — print HTML CSS
- `src/pages/HRPg.jsx` — multiple inline CSS strings

The CSS font-family chain across all these is now:
```css
font-family: 'Markazi Text', 'Cairo', serif;
```

Cairo retained as a runtime fallback so that if a user's first print is offline (Markazi Text not yet cached), Cairo (still in the Google Fonts CSS `<link>`) takes over and Arabic still renders correctly.

## Where Cairo stays

Anywhere not a print/PDF output:
- `src/utils/popups.js` — toast, ask, askInput, askForm popups
- `src/components/LoginScreen.jsx`, `ConfirmPage.jsx`, `CustomerPortalPage.jsx`, `WorkshopConfirmPage.jsx`, `WorkshopPortalPage.jsx`, `ErrorBoundary.jsx`
- `src/pages/SettingsPg.jsx` (font picker UI keeps Cairo as a value)
- `src/App.jsx` (the QR-label settings dropdown — user can still pick Cairo for thermal labels)
- `src/utils/print.js` `_GOOGLE_FONT_URLS_QR` map — still includes Cairo as a per-label fontFamily option (settings-driven)

## Font sizes adjusted

Markazi Text is a serif and tends to render slightly smaller than Cairo at the same pixel size. To keep visual weight + readability, `PRINT_CSS` sizes were bumped:
- `body` font-size: 12px → 14px
- `h2`: 15px → 17px
- `h3`: 13px → 15px
- `td`: 11px → 13px
- `.hdr-info`: 11px → 12px
- Other minor adjustments throughout

## Test plan

### jsPDF PDFs
1. Hard reload (Ctrl+Shift+R).
2. CustDeliverPg → group-print → tick `📎 إرفاق نسخة PDF` → send.
3. Network: `/fonts/MarkaziText-Regular.ttf` + `/fonts/MarkaziText-Bold.ttf` → 200 OK.
4. PDF received: text should be in Markazi Text style (modern serif).

### Browser print + Save as PDF
1. Open any page that has a print button (e.g. CustDeliverPg → "🖨 طباعة الجدول", a sales report, an invoice).
2. Print preview should show Markazi Text body + headers.
3. Save as PDF → font is preserved.

### Visual style
1. Markazi Text is a serif (not calligraphic like Amiri, not modern sans like Cairo). It has gentle serifs and looks document-like.
2. If the user wants a different look, the choice is reversible — just swap the bundled TTFs and update CSS. Mada (geometric sans) and Reem Kufi (geometric Kufic) are alternatives if Markazi feels too formal.
