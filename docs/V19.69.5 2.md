```
SyntaxError: Unexpected token '*' at compileSourceTextModule
```

الـreal root cause: `*/5 * * * *` (cron pattern) **داخل** `/* */` block comment في `api/automation-tick.js:5`. الـ`*/` بيـclose الـcomment بدري، الـparser يحاول يـparse الباقي كـcode → fails → Vercel runtime يرد generic 500 بدون body.

V19.69.3 fix (cross-folder import) ما كانش relevant — الـbug تاني خالص. الـbuilder copy في `api/_buildDailyReport.js` لسه موجود كـdefense-in-depth (ميـضرش).

## Manual setup needed (one-time)

If deploying to a fresh Vercel project, follow `docs/V19.69.md` Step 1-4. If migrating an existing deploy:

1. **Vercel env**: ensure `AUTOMATION_TICK_SECRET` is set with a strong random value (32+ chars). Redeploy after change.
2. **VPS crontab**: SSH to the bridge VPS, add the crontab line (see operations section above), set the same secret.
3. **Verify**: `curl -fsS .../api/automation-tick -H "Authorization: Bearer $SECRET"` from the VPS should return 200 with `triggerSource: "cron"`. If 401 → secret mismatch.
