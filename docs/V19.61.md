## 🐛 V19.61 الـBugs

### Bug #1: Race بين listener fire والـsale
1. listener customers يـfire → `setPartitionedData(prev => ...customers: [35])`
2. State queued في React (لسه ما committed)
3. `partitionedDataRef.current` لسه = old value (useEffect ما اشتغلش بعد commit)
4. **اللحظة دي** المستخدم يضغط "تأكيد البيع"
5. upConfig يقرأ `ref.customers = []` (stale!)
6. `newPart.customers = []` → `setPartitionedData(newPart)` → state.customers = []
7. القائمة تختفي

### Bug #2: Listener error wipes cache
1. سعر sale يـtrigger setDoc على factory/config
2. كل listeners تعيد تقييم Firestore rules (لأن `getRole()` بيقرأ factory/config)
3. لـmillisecond، rule eval ممكن يفشل transiently
4. customersDocs listener يـerror
5. Error handler: `firstFires.customers = true; rebuild()`
6. rebuild → `flatten(empty Map) = []` → cache wiped

### Fix #1: Sync ref update داخل setState callback
```js
setPartitionedData(prev => {
  const next = {...prev};
  for (const f of PARTITIONED_FIELDS) {
    if (firstFires[f]) {
      next[f] = flatten(...);
    }
  }
  partitionedDataRef.current = next;  // ← V19.61: sync update
  return next;
});
```
