## 2. كيف يعمل نظام المستندات الموحّد (مهم للتعديلات الجاية)

كل مستندات المبيعات/المشتريات بتعرض بنودها من **مصدر واحد**:

- **`src/utils/docColumns.js`** — طبقة العرض النقية:
  - `buildDocColumns(items, opts)` → `{ rows, totals }`. كل صف فيه:
    `{code, name, unit, qty, price, subBefore, discountPct, discount, subAfter}`.
    بيوزّع **الخصم الكلي** على الصفوف بالتناسب (آخر صف يمتص فرق القرش).
    `opts.headerDiscountPct` (عرض/أمر) أو `opts.headerDiscountAmount` (فاتورة).
  - `docColumnsHTML(items, opts)` → جدول HTML للـ PDF (٩ أعمدة) + إجماليات + تفقيط.
  - **مُختبَر:** `src/utils/__tests__/docColumns.test.js` (الضمانة الذهبية:
    مجموع «بعد الخصم» للصفوف = إجمالي المستند بالظبط).
- **`src/components/DocItemsTable.jsx`** — جدول البنود على الشاشة (٩ أعمدة)
  + مكوّن `DocTotals` (الإجماليات + نسبة الخصم جنب «إجمالي الخصومات» + التفقيط تحت).
- **`src/utils/tafqit.js`** — `tafqitEGP(amount)` → «فقط ... جنيهاً مصرياً لا غير».
  مُختبَر: `src/utils/__tests__/tafqit.test.js` (١٤ اختبار).

**القاعدة:** أي تعديل في شكل/أعمدة/خصم/تفقيط أي مستند → عدّل **هنا** (مكان
واحد) فينطبق على الستة. **لا تكرّر** جدول بنود في أي مودال/طباعة.

**المستندات الموصولة بالنظام:**
- عرض السعر: `src/components/sales/QuotationDetailModal.jsx` (تفاصيل + `buildQuoteHTML` للطباعة)
- أمر البيع: `src/components/sales/SalesOrderDetailModal.jsx` + `src/utils/sales/docPrint.js`
- فاتورة (مبيعات+مشتريات): `src/pages/SalesInvoicesPg.jsx` (`InvoiceDetailModal`) + `src/utils/printInvoice.js`
- RFQ: `src/components/purchase/RfqDetailModal.jsx` (الطباعة `printRfq` لسه نموذج طلب أسعار فاضي — مقصود)
- أمر الشراء: `src/pages/PurchasePg.jsx` (`viewPo` + `printPo` + `savePo`)

**نموذج بيانات الخصم:**
- بند: `discountType: "pct"|"amount"` + `discountValue` (محرّر مشترك `DocLineEditor`).
- رأس المستند: `discountPct` (عرض/أمر/RFQ/PO) أو `discount`/`discountValue` (فاتورة).
