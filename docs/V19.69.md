## السياق

V19.68 ضافت Daily Report builder + manual "Send Test Now". V19.69 يـwire الـscheduling:
- VPS cron يـpings Vercel API كل 5 دقايق
- API endpoint يـcheck Cairo time → builds report → posts to bridge
- Idempotent: مش يبعت مرتين في نفس اليوم
- يشتغل **والبرنامج مقفول** (الـVPS مش الـclient)

## ⚙️ Setup Steps (لازم تنفذهم بنفسك على الـVPS)

### Step 1: Vercel Environment Variable

في Vercel dashboard → Project → Settings → Environment Variables، ضيف:

```
AUTOMATION_TICK_SECRET = <generate a long random string, e.g. 32 chars>
```

مثال على random secret: `XK7p9mQ2nL5vR8sT1wY4cE6gA3bF0hJ9`

ثم redeploy الـapp (Vercel سيـpick up الـvar).

### Step 2: SSH للـVPS و إعداد Crontab

```bash
ssh root@77.237.235.160      # أو أي user عندك
crontab -e
```

ضيف السطر التالي (استبدل `YOUR_SECRET` بالـAUTOMATION_TICK_SECRET نفس اللي حطيته في Vercel، و `your-app.vercel.app` بـdomain الـapp بتاعك):

```
*/5 * * * * curl -fsS -X GET "https://your-app.vercel.app/api/automation-tick" -H "Authorization: Bearer YOUR_SECRET" >> /var/log/clark-automation.log 2>&1
```

**شرح الـcrontab:**
- `*/5 * * * *` — كل 5 دقايق
- `-fsS` — fail silently on 4xx/5xx لكن يـlog، يـshow errors لـcron mail
- `>> /var/log/clark-automation.log` — append output for debugging

### Step 3: التأكد من timezone الـVPS

الـAPI نفسها بتـuse Africa/Cairo (UTC+2) للـcomparison بصرف النظر عن timezone الـVPS، فـ**مش لازم تغيّر timezone الـVPS**. لكن لو حابب الـcron logs بـCairo time:

```bash
sudo timedatectl set-timezone Africa/Cairo
```

### Step 4: اختبار الـtick يدوياً

من جوّه الـVPS:

```bash
curl -X GET "https://your-app.vercel.app/api/automation-tick" \
  -H "Authorization: Bearer YOUR_SECRET"
```

الـreturn يفترض يكون JSON زي:

```json
{
  "ok": true,
  "cairoTime": "2026-05-05 14:23",
  "actions": [{"type":"dailyReport","due":false,"reason":"before-scheduled"}],
  "errors": []
}
```

لو الـauth غلط: `{"ok":false,"error":"Unauthorized"}` (status 401).
لو في error في firestore: status 503 + الـerror.

### Step 5: تأكد من الـconfig في الـapp

افتح Automation page:
- في الـheader، الـ"VPS Cron" indicator لازم يقول "🟢 نشط" بعد أول tick (5 دقايق max)
- "آخر tick" يحدّث كل refresh
- "الإرسال القادم" يعرض الـtime الـnext run في Cairo

## 🔧 الـTechnical decisions

### لماذا Cron على الـVPS مش Vercel Cron؟
- **Vercel Hobby plan = 1 cron/day max**. ده غير كافي لـ5-min ticks.
- **VPS cron مجاني + موثوق** — الـVPS أصلاً up 24/7 للـbridge.
- **Single source of truth** للـscheduling — كل الـauto stuff يبدأ من نفس الـVPS.

### لماذا 5-min granularity؟
- Trade-off بين دقة الـschedule و عدد الـticks (Vercel function invocations)
- لو الـscheduled time = 08:00، الـtick بين 08:00 و 08:05 يـtrigger الـsend
- 5 min latency مقبولة لـdaily reports (مش real-time alerts)

### لماذا "≥ scheduled time" مش "exact match"?
- لو الـVPS down عند 08:00 (reboot، crash)، الـsend يـcatch up أول tick بعد ما يرجع
- الـ"already sent today" check يضمن ميـبعتش مرتين

### لماذا Africa/Cairo hardcoded?
- الـuser في مصر، factory في مصر، الـrecipients كلهم في مصر
- ميستخدمش DST (constant UTC+2 منذ 2020)
- لو في expansion لبلد ثاني، نـadd `data.automation.timezone` field لاحقاً

## 🛡️ Security Notes

- الـ`AUTOMATION_TICK_SECRET` shared between Vercel env و VPS crontab
- الـsecret لازم يكون قوي (32+ chars random)
- لو اتسرب: rotate في Vercel + update crontab
- الـbridge على الـVPS protected by `AUTH_TOKEN` (separate)
- Firestore reads scoped to admin SDK (server-side)

## 📊 Monitoring & Debugging

**في الـapp:**
- Automation tab → "VPS Cron" indicator (🟢/🟡)
- "آخر tick" timestamp
- History tab — كل send مع status

**على الـVPS:**
```bash
# آخر 50 سطر من الـcron log
tail -50 /var/log/clark-automation.log

# تابع الـcron live
tail -f /var/log/clark-automation.log

# تأكد إن الـcron entry موجود
crontab -l
```

**لو الـtick failing:**
- الـlog هيـshow الـHTTP status
- لو 401 → secret mismatch
- لو 503 → check Vercel logs (Firestore unreachable)
- لو 502/504 → check bridge URL في `data.campaignBridge.url`

## 📝 خطة الاختبار

### Test 1: Manual API ping
```bash
curl -v "https://your-app.vercel.app/api/automation-tick" \
  -H "Authorization: Bearer YOUR_SECRET"
```
- 200 OK + JSON يعني الـendpoint شغال
- 401 يعني secret غلط
- 503 يعني Firestore أو bridge مش accessible

### Test 2: Crontab شغال
بعد ~10 دقايق من إعداد الـcrontab، افتح Automation tab. الـ"VPS Cron" indicator لازم يبقى 🟢 نشط.

### Test 3: Schedule trigger
1. اضبط الـtime في Daily Report لـ"تاريخ الآن + 6 دقايق" (مثلاً لو الساعة 14:00، اضبطها 14:06)
2. Save
3. انتظر 6-10 دقايق
4. الواتس هتجي
5. تأكد إن الـhistory عنده entry بـ`source: "scheduled"`

### Test 4: Idempotency
1. بعد ما الـscheduled run يخلص، انتظر tick آخر (5 دقايق)
2. تأكد إن مفيش رسالة ثانية
3. الـHistory tab مفيش entry جديدة

### Test 5: Bridge offline scenario
1. أوقف الـbridge على الـVPS مؤقتاً
2. Trigger schedule
3. الـHistory entry لازم تكون ❌ مع الـerror
4. شغّل الـbridge، انتظر next day، تأكد إن الـmessage بيـsend
