## الإصلاحات

### 🚨 CRITICAL #1: Manager-to-admin self-promotion

**File:** `firestore.rules` (split factory/config from factory/{others})

**قبل:**
```js
match /factory/{docId} {
  allow read:  if isAnyUser();
  allow write: if isManagerPlus();/* manager can write ALL fields */
}
```

أي manager → DevTools → `setDoc(doc(db,"factory","config"), {users: {[myUid]:'admin'}}, {merge:true})` → instant admin.

**بعد:**
```js
function affectsSensitiveFields() {
  return resource != null
      && request.resource.data.diff(resource.data).affectedKeys()
         .hasAny(['users','usersList','permissions','customRoles']);
}

match /factory/config {
  allow read:   if isAnyUser();
  allow create: if isAdmin();
  allow update: if isAdmin() || (isManagerPlus() && !affectsSensitiveFields());
  allow delete: if isAdmin();
}

match /factory/{docId} {
  allow read:  if isAnyUser();
  allow write: if isManagerPlus();
}
```

manager لسه يقدر يحفظ settings عادية (factoryName، invoiceSettings، etc.) لأن affectedKeys() مش هيشمل الحقول الحساسة. لكن أي محاولة لتعديل users/permissions/customRoles/usersList → blocked.

### 🚨 CRITICAL #3: Storage rules path-prefixed scoping

**قبل:** `match /{allPaths=**}` — أي authed user يكتب/يحذف أي ملف.

**بعد:**
- `invoices/**` → sales + purchase scope (create/update)، manager+ delete
- `orders/**`, `seasons/**` → manager+
- `templates/**`, `campaigns/**` → sales scope
- `logos/**` → admin only
- `qr/**`, `attachments/**` → manager+
- `temp/**` → any authed (auto-cleanup expected)
- Default catchall = **deny**

كمان MIME-type guard: images/PDF/office docs only. Executables blocked. + delete restricted to manager+ across the board.

### 🛡️ Portal HMAC expiry — 90 days

**قبل:** Customer/Workshop signatures كانوا HMAC على الـID فقط، مفيش timestamp. الـlink يفضل valid forever.

**بعد:** Sign endpoints تـembed `t=<unix-seconds>` في الـHMAC payload. Verify endpoint يـreject لو `now - ts > 90 * 24 * 3600`. الـold unbounded format لسه مقبول للـlinks المُنشأة قبل V19.64 (backward compat).

```js
// Before
url = `?p=c&i=${custId}&s=${signCustomerId(custId)}`

// After (V19.64)
const ts = Math.floor(Date.now()/1000).toString();
url = `?p=c&i=${custId}&t=${ts}&s=${signCustomerIdWithTs(custId, ts)}`
```

### 🛡️ HR collections — payroll_verifier write removed

**قبل:** `hrWeeksDocs` write = isHRRole (شامل verifier). الـverifier role صلاحياته QR-scan only، فميـلزمش write على hrWeeksDocs.

**بعد:** `hrWeeksDocs` write = `isHRWriter` (admin/manager/payroll_accountant — verifier excluded). الـverifier لسه يقدر يكتب في `hrLogDays` لـaudit-log entries. الـcurrent verify-flow في الـapp بيـwrite على hrWeeksDocs.receipts — TODO V19.65: نقل عبر admin-SDK API endpoint.

### 🛡️ auditDays + migrationLog writes tightened

- `auditDays` write: any authed → **manager+** (viewer لا يقدر يـpoison الـaudit trail)
- `migrationLog` write: any authed → **manager+** (الـboot-time migrations بتـrun في manager context)

### 🛡️ AI endpoint hardened

**قبل:** `/api/ai` كان مفتوح للنت كله بدون auth. cost abuse + prompt injection + system prompt exfil.

**بعد:**
- Require Firebase ID token (any authed user)
- Per-UID rate limit: 30 req / 5 min (in-memory)
- Body size cap: 50KB messages، 4KB system prompt
- Optional `AI_SYSTEM_PROMPT` env فيخلي الـsystem prompt server-fixed
- Optional `AI_ALLOWED_ORIGIN` env للـCORS

### 🛡️ odoo-sync hardened

**قبل:** SSRF proxy — أي حد يـfetch لأي URL مع credentials.

**بعد:**
- Require admin Firebase token
- Optional `ODOO_ALLOWED_HOSTS` env (CSV) لـallowlist
- Optional `ODOO_ALLOWED_ORIGIN` env للـCORS

### 🔧 setCors helper — origin allowlist

`api/_firebase.js setCors(res, req)` بقى يدعم `API_ALLOWED_ORIGIN` (single) أو `API_ALLOWED_ORIGINS` (CSV). Fallback `*` (back-compat).

## 📋 Manual Steps After Deploy (CRITICAL)

1. **Deploy الـrules:**
   ```bash
   firebase deploy --only firestore:rules,storage:rules
   ```

2. **Disable Firebase signup self-signup:**
   - Firebase Console → Authentication → Sign-in method
   - Email/Password → Disable "User registration"
   - أنشئ users جدد فقط من الـapp (admin → SettingsPg → "إضافة مستخدم")

3. **Set Vercel environment variables:**
   ```
   BOOTSTRAP_ADMIN_UID=<your-admin-uid>
   API_ALLOWED_ORIGIN=https://your-vercel-domain.vercel.app
   AI_ALLOWED_ORIGIN=https://your-vercel-domain.vercel.app
   ODOO_ALLOWED_ORIGIN=https://your-vercel-domain.vercel.app
   ODOO_ALLOWED_HOSTS=your-odoo-host.com
   AI_SYSTEM_PROMPT=<optional fixed system prompt>
   ```

4. **اختبر الـ-rules قبل ما الـusers يـreport bugs:**
   - Login كـmanager → افتح DevTools → جرّب `setDoc(doc(db,"factory","config"), {users:{[uid]:'admin'}}, {merge:true})` → لازم يفشل بـpermission denied
   - Login كـviewer → جرّب `getDocs(collection(db,"treasuryDays"))` → لازم يفشل
   - افتح customer portal URL قديم (بدون `?t=...`) → لازم يفتح (backward compat)
   - عمل URL جديد عبر sign endpoint → فيه `&t=...` → يفتح
   - عمل URL جديد، عدّل `t` لـ100 يوم في الماضي → "رابط منتهي الصلاحية"
