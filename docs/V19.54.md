## 🔍 الـRoot Cause — bug جذري حرج

في `App.jsx`، `upConfig` كان بيقرا `configDoc` من React closure:
```js
const upConfig = useCallback(fn => {
  ...
  const prev = configDoc || {};  // ← stale closure!
  ...
});
```

### السيناريو الخطر (bulk post)
- المستخدم اختار 5 فواتير draft
- `runBulkPost` يعمل loop:
  ```js
  for(const item of selectedItems){
    await postOne(item, { silent: true });
  }
  ```
- Iteration 1 → `upConfig(d => ...)` → `prev = configDoc` (snapshot)
  - mutates next.salesInvoices[0].status = "posted"
  - `setConfigDoc(stripped)` — optimistic update
- **قبل ما React يعيد render**، Iteration 2 يبدأ
- Iteration 2 → `upConfig(d => ...)` → `prev = configDoc` **(نفس الـsnapshot القديم!)**
  - بيشوف invoice 0 لسه status="draft"
  - mutates next.salesInvoices[1].status = "posted"
  - `setConfigDoc(stripped)` — **يستبدل state كله** بـ`prev` المُعدّل
  - النتيجة: invoice 0 رجع draft! invoice 1 posted

كل iteration بتـoverwrite الاللي قبلها بـstale state. **النتيجة النهائية**: آخر iteration بس بتفضل، الباقي يرجعوا draft.

### الـbug ده كان موجود من V16.80 (2025)
أي 2 actions متلاحقين في نفس الـtick (sequential mutations) كانوا معرّضين. لكن لاحظناه دلوقتي بسبب bulk-post اللي بيعمل 5+ في الثواني.

## ✅ الإصلاح

### 1. استخدام `configDocRef.current` بدل closure
الـref ده موجود من V19.48 لكن كان مستخدم في listener detection فقط. V19.54 خلّاه source of truth للـoptimistic state:

```js
// قبل V19.54:
const prev = configDoc || {};   // stale!

// بعد V19.54:
const prev = configDocRef.current || configDoc || {};
```

### 2. تحديث الـref synchronously بعد `setConfigDoc`
الـuseEffect اللي بيـsync الـref بيشتغل بعد render. مش كافي لـsync loops:

```js
setConfigDoc(stripped);
configDocRef.current = stripped;   // ← V19.54: sync update
```

دلوقتي iteration N+1 بتشوف iteration N's optimistic state فوراً.

### 3. نفس الإصلاح على `upSales` + `upTasks`
الـ3 helpers الـcore عندهم نفس النمط. كلهم اتصلحوا.

| Helper | Ref المُستخدم | فايل |
|---|---|---|
| `upConfig` | `configDocRef.current` | App.jsx |
| `upSales` | `salesDocRef.current` | App.jsx |
| `upTasks` | `tasksDocRef.current` | App.jsx |

- اسم الفاتورة الحالية live
