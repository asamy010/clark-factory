## 🐛 Bug #1: قائمة العملاء تظهر فاضية (0)

**التقرير**: "فتحت البرنامج لقيت قائمة العملاء مختفية. قفلت وفتحت تاني ظهرت."

### الـRoot Cause
في V19.57 عملنا master data byId. الـmerge logic في `config` useMemo كان:
```js
if (partitionedLoaded && configDoc[PARTITIONED_FLAG_V1957]) {
  for (const f of PARTITIONED_FIELDS_V1957) merged[f] = partitionedData[f] || [];
}
```

**شرطين لازم يتحققوا**:
1. `configDoc[FLAG] = true` (migration خلصت)
2. `partitionedLoaded = true` (كل الـ8 listeners اتزرعوا واشتغلوا)

**الـbug**: لو **listener واحد** بطيء أو فشل (slow network, multi-tab race, transient permission error) → `partitionedLoaded` يفضل `false` → الـmerge لا يحصل أبداً → `data.customers = undefined`.

ده كان معقّد على سيناريوهات multi-user (المستخدم قال "أكتر من مستخدم في نفس الوقت من أماكن مختلفة").

### الـFix
- شيلت `splitLoaded`/`partitionedLoaded` gates من الـuseMemo — كل listener يـfill independently
- partitionedData/splitData تبدأ فاضية، تتعمر بمجرد أي listener fire
- الـloaded flags لسه بتمنع الـwrites (upConfigTx etc.) — ده الصح
- إضافة hydrate من localStorage على mount — المستخدم يشوف آخر بيانات معروفة فوراً (zero loading flash)
- الـsnapshot بقى يشمل كل master data (8 entities)

### النتيجة
لو listener بطيء → user يشوف الـdata من الـcache فوراً + listener يحدّث لما يخلص. مفيش "0 عملاء" أبداً.

## 🐛 Bug #2: البحث في البيع السريع ميـجبش الموديل

**التقرير**: "بكتب رقم الموديل في البحث مش بيظهر، مع إنه في الجدول والعميل مستلمه."

### الـRoot Cause
في الـquick-sale popup، الـdropdown filter:
```js
linkedSess.modelIds.includes(m.id)  // m.id = order id
```

**الـbug**: نفس الـmodelNo ممكن يكون ليه **order ids مختلفة**:
- Re-cut → order جديد
- Season جديد → order جديد
- تعديل الكميات → order جديد

الـsession's `modelIds` فيها order id قديم (لما الـsession اتعملت). الـ`stockModels` فيها order id الجديد (اللي عنده stock available). الـ`includes` تفشل لأن الـids مختلفة → الـmodel يختفي من الـdropdown.

### الـFix
بدل المقارنة بـorder id، نقارن بـ**modelNo**:
```js
const sessModelNos = new Set();
for (const oid of (linkedSess.modelIds || [])) {
  const o = orders.find(x => x.id === oid);
  if (o && o.modelNo) sessModelNos.add(o.modelNo);
}
baseList = stockModels.filter(m => m.avail > 0 && sessModelNos.has(m.modelNo));
```

طُبّق في **3 مواضع**:
1. الـdropdown filter
2. Manual add path
3. QR scan path

## 🛡️ الـArchitecture الجديد

```
قبل V19.59:
  configDoc loaded → useMemo waits for partitionedLoaded → data flows
  لو listener stalled: data forever undefined

بعد V19.59:
  configDoc loaded → useMemo merges available partitionedData → data flows
  لو listener stalled: data uses last-known cache or empty array
                       (recovers as soon as listener fires)
  Writes still gated: upConfigTx refuses if listeners not ready (correct)
```
