# V19.74.0 — AI Agent: "Save Changes" pattern (no more instant writes)

**Date:** 2026-05-06
**Type:** UX + Architectural

---

## ملخص

User feedback: *"عاوز اي تعديل في اي مكان بعدها اضغط حفظ التغييرات ويشوف التعديلات بلاش التعديل المباشر اللحظي في اي حاجة ده بيعمل مشاكل"*

كل tabs الـ AI Agent كانت بـ تـwrite على Firestore لحظياً مع كل keystroke/toggle/click. ده كان بـ يـcause:
- **Cost** — كل حرف = Firestore write
- **Flicker** — الـ listener بـ يـfire ويـsnap الـ UI لحظياً
- **Race conditions** — لو 2 admins بـ يعدّلوا، الـ آخر write بـ يـwin
- **Edits lost** — لو الـ network بـ يـfail mid-edit، البيانات تختلط

الـ Fix: **draft + Save Changes pattern**.

## How it works

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   data.aiAgent (server)                                 │
│        │                                                │
│        ▼ (deep clone on mount + sync if !dirty)         │
│   draft (local React state)                             │
│        │                                                │
│        ▼ (every edit mutates the draft, NOT server)     │
│   updateAgent(mutator) → setDraft(deepClone+mutate)     │
│        │                                                │
│        ▼                                                │
│   dirty = true → sticky save bar appears                │
│        │                                                │
│        ▼ (user clicks 💾)                               │
│   saveChanges() → upConfig(d => d.aiAgent = draft)     │
│   dirty = false → bar hides                             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## What changed

### Page-level draft (AIAgentPg)

```js
const [draft, setDraft]  = useState(() => deepClone(data.aiAgent));
const [dirty, setDirty]  = useState(false);

useEffect(() => {
  if (!dirty) setDraft(deepClone(data.aiAgent));  // re-sync ONLY if no local edits
}, [JSON.stringify(data.aiAgent)]);

const updateAgent = (mutator) => {
  setDraft(prev => { const next = deepClone(prev); mutator(next); return next; });
  setDirty(true);
};

const saveChanges = () => {
  upConfig(d => { d.aiAgent = deepClone(draft); });
  setDirty(false);
};

const discardChanges = async () => {
  if (!await ask("التراجع عن كل التغييرات غير المحفوظة؟")) return;
  setDraft(deepClone(data.aiAgent));
  setDirty(false);
};
```

All 5 sub-tabs that were calling `updateAgent` (Personality / Schedule / Tools / FAQs / Funnel) automatically benefit — no changes to their code needed because the function signature stayed the same; only the body changed.

### Sticky save bar in header

Visible **only when dirty**. Position: sticky top:0, z-index:50. Orange/amber gradient with warning icon, descriptive copy, and 2 buttons:
- **↩️ تراجع** — discards (with confirmation prompt)
- **💾 حفظ التغييرات** — commits to Firestore

When dirty=false, the bar doesn't render at all (no permanent UI overhead).

### Power toggle now drafted

The "▶️ تشغيل / 🛑 إيقاف" button in the header used to write immediately. Now it just flips `draft.enabled` — the admin must Save to apply. The status pill ("شغّال" / "موقوف") reflects the draft state.

### CustomerFullProfileModal

Same pattern but per-modal-instance:

```js
const [draft, setDraft] = useState(() => deepClone(customer));
const [dirty, setDirty] = useState(false);

const handleClose = async () => {
  if (dirty && !await ask("في تغييرات غير محفوظة. تأكيد الإغلاق؟")) return;
  onClose();
};
```

Clicking the backdrop or ✕ button checks for unsaved changes before closing. Save button only appears in the footer when dirty.

### Toast wording

Per-action toasts that used to say *"تم الحفظ"* now say *"تم التحديث (اضغط حفظ التغييرات)"* — accurate to the new model. The actual *"✓ تم حفظ كل التغييرات"* toast fires only after the Save button.

## Why this matters even more for Phase D

When the agent backend (Phase D) starts running, it will continuously write to:
- `aiAgentAnalytics/{YYYY-MM-DD}` — daily aggregates
- `aiAgentConversations/{conversationId}` — every WhatsApp exchange
- `aiAgentSuggestions/{suggestionId}` — pending observations + transitions

If admin's UI was still doing instant writes on `data.aiAgent` (config doc), there'd be heavy contention on the `factory/config` document. Now:
- Admin writes go to `data.aiAgent` only on explicit Save (rare)
- Backend writes go to separate `aiAgent*` collections (independent docs)
- No write conflicts between the two

## Backward compatibility

- All existing edits continue to work — the function `updateAgent` keeps the same signature
- The draft is deep-cloned on every mutation so server data is never accidentally mutated by reference
- If the user navigates away with unsaved changes, the changes are lost (intentional — the whole point is admin must commit explicitly)
