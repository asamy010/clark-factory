## 🧩 ميزة المحافظ الإلكترونية — الحالة التفصيلية (للاستكمال)

**كله في `src/pages/TreasuryPg.jsx`. القرار المعماري: المحفظة = treasuryAccount (مش نظام موازي).**

**نموذج البيانات** (على الـ account object في `cfg.treasuryAccounts`):
```js
{ id, name, ownerEmail, type:"wallet",
  walletNumber, icon, image,           // عرض/مراقبة
  balanceCap, monthlyWithdrawCap,      // حدود (افتراضي 200000)
  tiers:[{from, to, pct}] }            // عمولة نسبة % (to=null=مفتوح؛ legacy fee=ثابت)
```

**الدوال المفتاحية (TreasuryPg.jsx):**
- `computeWalletFee(w, amount)` — نسبة % من الشريحة المطابقة (مع fallback لـ fixed `fee`).
- `walletCard(w)` — البطاقة المشتركة (تاب المحافظ + تاب الحساب).
- `walletMonthOut` (useMemo) — مجموع سحب الشهر الحالي لكل حساب (مفتاح YYYY-MM، بيتجدد يوم 1).
- `buildWalletFields()` — بناء بيانات المحفظة من الفورم. `addAccount(typeArg)` — يأخذ type صريح (مايعتمدش على state). `editAccount(a)` — hydrate.
- `saveTxWithLimits()` — **wrapper قبل saveTx** (زر الحفظ بيناديه) — فحص الحدود (تحذير + تجاوز مدير). **`saveTx` نفسها ماتلمستش.**

**الاستلام/الصرف/التحويل/السجل:** شغّالين تلقائياً (المحفظة account → بتظهر في كل dropdowns).

**العمولة:** hook في `saveTx` (داخل branch إنشاء حركة جديدة، **محمي بـ `txType==="out" && wallet && fee>0`**) → entry "عمولة محفظة" منفصل + auto-post. preview في فورم الحركة.

**الحدود:** `saveTxWithLimits` — حد الرصيد على الاستلام (in)، حد السحب الشهري التراكمي على الصرف (out). تحذير للكل، تجاوز للـ admin/manager، بلوك لغيرهم.
