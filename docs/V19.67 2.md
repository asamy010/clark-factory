## الإصلاحات

### 🚨 #1 + #2: ترحيل القيود — Bug fix + Lock UI

**الـRoot cause** (`src/utils/accounting/backfill.js:68`):

```js
// قبل V19.67:
await postEntry({...entry, coa: args[args.length-2], createdBy});
```

ده بيفترض إن آخر argument في كل builder هو `rules`، فالـ`coa` يقع عند `args.length-2` (penultimate). صحيح لـbuilders زي:
- `buildSaleEntry(delivery, customer, order, coa, rules)` — 5 args، coa = args[3] ✓
- `buildWorkshopReceiveEntry(rcv, ws, order, wd, coa, rules)` — 6 args، coa = args[4] ✓

لكن **خاطئ** لـbuilders اللي تأخذ `config` كـlast arg:
- `buildCustomerPaymentEntry(payment, customer, coa, rules, config)` — 5 args، coa = args[2] (مش args[3])
- `buildWorkshopPaymentEntry(...)` — same shape
- `buildHrEntry(...)` — same shape
- `buildTreasuryEntry(...)` — same shape

`args[args.length-2]` لـ5-arg builder يـpoint على `rules` object (مش coa). الـpostEntry يـcall `validateLines(lines, coa)` → `getAccount(coa, id)` → `(coa||[]).find(...)`. لو `coa` بقى rules object (truthy لكن مش array)، الـ`||[]` ميـtriggerش، والـobject مش له `.find` → **crash**.

**الـFix**: pass `coa` explicitly من الـouter scope:

```js
// V19.67:
async function _safePost(s, builder, type, args, createdBy, coa){
  ...
  await postEntry({...entry, coa, createdBy});
}

// caller:
await _safePost(stats, t.builder, t.type, t.args, createdBy, coa);
```

الـ536 customerPay errors المتوقع تتحل دلوقتي.

### 🔒 الـModal Popup مع Progress + Lock

`src/components/accounting/AccountingSettingsTab.jsx`:

- **قبل**: progress inline في الـcard. الـuser يـclick على tab تاني → الـrun بيكمل في الـbackground، الـwrites بيخلصوا، لكن الـuser ميشوفش الـoutcome ولو في error مش هيعرف. بعض الـwrites قد تكون orphan.
- **بعد**: full-screen overlay modal بـz-index 99999 + `backdrop-filter:blur(3px)`. مفيش close button. الـuser ميقدرش يـnavigate. يـunlock تلقائياً بعد completion.
- زيادات: percentage badge في وسط الـbar (mix-blend-mode للـreadability)، animated emoji، warning message.

### 🔒 #3: قفل مدير النظام

`src/pages/SettingsPg.jsx:3138-3140`:

```jsx
// قبل V19.67:
<Sel value={u.role} onChange={...}>
  {effectiveRoles.map(...)}
</Sel>
<DelBtn onConfirm={...}/>
```

أي user بصلاحية `settings:edit` يقدر يـ-demote الـadmin، أو يحذفه. الـV19.64 Firestore rules بـenforce على الـDB، لكن الـUI ميـreflectش الـconstraint.

**الـFix** — لو `u.role === "admin"`:
- الـname column: badge "🔒 محمي" warning color
- الـrole column: read-only display "👑 مدير النظام" بدلاً من Sel
- الـactions column: '🔒' غير interactive (no DelBtn)

ده defense-in-depth — UI و DB layer متطابقين.

### 🧹 #4: تنظيف شاشة أوامر القص

`src/pages/DetPg.jsx:131-173`:

شيلت الـ4 KPI cards (إجمالي الأوامر، كمية القص، متأخر، الإنجاز) كاملة. الـcounts الفعلية لسه متاحة عبر:
- الـstatus chips أسفل الـsearch bar (الكل، متأخر، per-status)
- الـ`detSt==='⚠️'` filter لسه شغّال عبر chip "متأخر"

شيلت كذلك الـ`totalCut`/`totalDel`/`comp` calculations اللي ما كانتش بتُستعمل في غير الـcards.
