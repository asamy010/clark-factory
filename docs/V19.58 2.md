## 🎯 الـ3 features

### 1️⃣ إصلاح حرج للـauto-backup
**المشكلة المُكتشَفة**: الـ`comprehensiveBackup.js` (V18.62) كان بيـbackup الـcollections القديمة فقط:
- factory/config + sales + tasks
- treasuryDays + auditDays + hrLogDays
- hrWeeksDocs

**الـcollections اللي ضافناها في V19.49 → V19.57 (24+ collection) كانت مش متضمنة:**
- custPaymentsDays / supplierPaymentsDays / wsPaymentsDays / checksDays
- salesInvoicesDays / purchaseInvoicesDays / purchaseOrdersDays
- packagesDays / custDeliverySessionsDays / tasksDays / stickyNotesDays / inventoryAuditsDays
- stockMovementsDays / purchaseReceiptsDays / treasuryTransfersDays / salesAuditsDays
- notificationsDays / userNotifStates
- customersDocs / suppliersDocs / workshopsDocs / employeesDocs / empDebtsDocs / generalProductsDocs / fabricsDocs / accessoriesDocs

لو حصلت كارثة → الـbackup كان هيـrestore النسخة القديمة بس!

**الإصلاح**:
- `SALES_SPLIT_COLLECTIONS` + `TASKS_SPLIT_COLLECTIONS` بقوا متضمنين في الـbackup loops
- `STANDALONE_COLLECTIONS` (userNotifStates) بقت متضمنة
- الـrestore + estimate functions اتحدّثوا
- النتيجة: backup يومي تلقائي يغطي **كل** الـ29 collection

### 2️⃣ Schema validation بـZod (WARN-only)

**ملفات جديدة**:
- `src/schemas/index.js` — Zod schemas لـ13 entity مهم (customers, suppliers, workshops, employees, empDebts, salesInvoices, purchaseInvoices, purchaseOrders, custPayments, supplierPayments, wsPayments, checks, treasury)
- `src/utils/validateData.js` — validator يقارن diff + يـlog + يحفظ آخر 100 خطأ

**التشغيل**: `validateDoc(prev, next, "config")` يتـcall بعد `fn(next)` في:
- `upConfig` (factory/config writes)
- `upSales` (factory/sales writes)
- `upTasks` (factory/tasks writes)

**WARN-only mode**: مش بيمنع الـwrite. الأخطاء بتظهر في:
- console.warn
- Settings → "آخر تحذيرات التحقق"

**الفايدة**: لو حد عمل refactor وبدأ يكتب shape غلط، هتشوف الأخطاء فوراً بدل ما تتراكم بدون لاحظ.

**Schemas permissive**: `passthrough()` — أي extra fields مسموحة. بنـenforce shape الـcore فقط.

### 3️⃣ Rollups engine

**ملف جديد**: `src/utils/rollups.js` — 4 pure functions:

```js
computeFinancialRollup(data, {from, to})
// → { totals, counts, customers[], suppliers[] }

computeMonthlyRollup(data, "2026-04")
// → financial rollup for one month

computeCustomerStatement(data, custId, {from, to})
// → { customer, invoices, payments, checks, totals }

computeSupplierStatement(data, supplierId, {from, to})
// → { supplier, invoices, payments, checks, totals }
```

**الفايدة**:
- مكان واحد للـaggregation logic بدل ما كل صفحة تعيد كتابتها
- consistent بين DashPg/RepPg/AccountingPg/CustDeliverPg/PurchasePg
- pure functions — caller يستخدم `useMemo` للـcaching
- جاهز لاستخدام في الصفحات اللي محتاجة tuning للـperf (V19.59+)
