## Issue #1: Conversation persistence

### Backend (clark-ai-agent v1.0.1-phase2)

ملف جديد `src/agent/conversationLog.js`:
```js
export async function persistTurn(entry) {
  const db = getFirestore();
  const doc = { ...entry, at: ..., dayKey: cairoDayKey(), ... };
  if (entry.idempotencyKey) {
    await db.collection("aiAgentConversations").doc(entry.idempotencyKey).set(doc, { merge: true });
  } else {
    await db.collection("aiAgentConversations").add(doc);
  }
}
```

`orchestrator.js` بـ يستدعي `persistTurn()` بعد كل turn (success / canned / test-mode-rejected). webhook بـ يمرر `messageId` للـ orchestrator → بـ يستخدم كـ doc ID فالـ webhook retries لا تـ duplicate.

Schema per doc:
```
wid, phone, customerId, customerName, stage,
userMessage, assistantReply,
toolsUsed[], iterations, durationMs,
usage { input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens },
at (ISO), dayKey ("YYYY-MM-DD" Cairo),
idempotencyKey, agentVersion, error,
skipped, canned, skippedReason
```

### Frontend (CLARK Logs tab)

`useAgentCollection("aiAgentConversations", q => orderBy("at","desc").limit(200))` — live subscription. Each Firestore doc = one turn (مش thread). الـ frontend بـ يـ group by wid لتظهر كـ threads.

UI features:
- Filter: ok / skipped / error
- بحث في wid + اسم + رسائل
- Card collapsed: name/wid + last message + status + duration + cost
- Card expanded: full thread (user + agent bubbles) + timestamps + toolsUsed + iterations + errors
- LID flagged with 🔒 emoji

## Issue #2: Dashboard analytics aggregation

### Backend

ملف جديد `src/agent/analytics.js`:
- `buildDailyAnalytics(dayKey)` — scans `aiAgentConversations` where dayKey == X and aggregates
- `rollUpDay(dayKey)` — writes the aggregate to `aiAgentAnalytics/{dayKey}`
- `rollUpTodayAndYesterday()` — convenience wrapper

`src/index.js` schedules:
- Hourly cron: `cron.schedule("0 * * * *", () => rollUpTodayAndYesterday())`
- Boot trigger: 5s after server starts (so dashboard refreshes after deploy)

Schema per `aiAgentAnalytics/{YYYY-MM-DD}`:
```
dayKey, turnsTotal, turnsSuccessful, turnsCanned, turnsSkipped, turnsFailed,
uniqueSenders, avgDurationMs,
toolUsage { tool_name: count }, stages { stage: count },
tokens { input, output, cacheRead, cacheWrite },
estimatedCostUsd, updatedAt, agentVersion
```

Pricing constants reflect Claude Haiku 3.5 (input $1, output $5, cache write $1.25, cache read $0.10 per 1M tokens). Update if `src/agent/llm.js` changes the model.

### Frontend (CLARK Dashboard tab)

Live subscription via `useAgentCollection("aiAgentAnalytics")`. Range selector (today/7d/30d) builds Cairo-day keys and indexes the docs. KPI cards:

Row 1: 💬 محادثات · ✅ ناجحة · 👥 عملاء فريدين · 💰 تكلفة USD
Row 2: 🤖 رد آلي · ⏭ مش في whitelist · ❌ فشل · ⚡ متوسط الزمن

Chart: محادثات vs test-mode-skipped per day.

Side panels:
- 🪙 token breakdown (input / output / cache write / cache read) → total USD
- 🛠 top 5 tools used in the period

## Issue #3: Recent Senders panel

### Backend

ملف جديد `src/agent/recentSenders.js`:
```js
export async function recordRecentSender({ wid, text, reason }) {
  const userPart = canonicalUserPart(wid);  // digits only
  const ref = db.collection("aiAgentRecentSenders").doc(userPart);
  // get-then-set: increment count, update lastSeenAt, snippet, etc.
}
```

`orchestrator.js` بـ يستدعي `recordRecentSender()` لما الـ `decideAvailability` يـ reject بـ `test_mode_not_whitelisted` أو `test_mode_canned`.

`firebase-security.js` ضافت `aiAgentRecentSenders` للـ `AGENT_OWNED_COLLECTIONS`.

Schema per doc (id = userPart):
```
wid, userPart, isLid, firstSeenAt, lastSeenAt, count, recentMessage, lastReason
```

### Frontend (CLARK Test Mode card in Schedule tab)

New `<RecentSendersPanel />` rendered inside the Test Mode card whenever `tm.enabled === true`.

Live subscription to `aiAgentRecentSenders` ordered by `lastSeenAt` desc, limit 20. Auto-filters out senders already in the whitelist (so the list shrinks naturally as you promote them).

Per row:
- 🔒 LID badge (if applicable)
- WID (monospace, full)
- count + "آخرها 5د" / "1س" / "yesterday" relative time
- Snippet of recent message (truncated to 200 chars)
- Click "+ للـ whitelist" → expands inline form with optional name input → click "ضيف" → updates `agent.testMode.whitelist[]` immediately
- Trash icon to delete from `aiAgentRecentSenders` (just hides — doesn't change agent behavior)
