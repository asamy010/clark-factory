# V19.75.0 — AI Agent Test Mode (Whitelist Gate)

**Date:** 2026-05-06
**Type:** Feature

User request: *"عاوز ازود في الاوبشن ان نعمل تيست مع ارقام معينة مش لايف يعني لحد مانوصل لمرحلة كويسة"*

## ملخص

Soft-launch protocol — `testMode` toggle. لما enabled:
- الـ Agent بـ يرد فقط على الأرقام في الـ whitelist
- الباقي يا بـ يستلموا رسالة "تحت الاختبار" يا silent
- **مفيش Claude call على الأرقام خارج الـ whitelist** (cost protection)

## What's new

### Schema (constants/index.js)

```js
testMode: {
  enabled: false,
  whitelist: [],/* {id, wid, label, addedAt} */
  outsideBehavior: "canned",/* canned | silent */
  outsideMessage: "أهلاً بحضرتك. النظام تحت اختبار محدود حالياً، الفريق هيرد عليك بكرة بإذن الله.",
}
```

### Header banner (always visible when test mode active)

   باقي الأرقام بـ تستلم رسالة "تحت الاختبار"

### Whitelist editor (top of Schedule tab)

- Master toggle (شغّال/موقوف)
- Per-entry rows: WID + optional label + delete
- Add form: phone (auto-formatted to `201XXX@c.us`) OR raw WA-ID (e.g. `46480236761325@lid`)
- Outside behavior selector: canned vs silent
- Editable canned message

### Backend availability gate (clark-ai-agent)

Order of checks in `decideAvailability({wid, now})`:
1. `cfg.enabled` — skip silent if false
2. **`testMode.enabled` + whitelist** ← NEW
3. Schedule mode (off/24x7/specific)

Whitelist match is **tolerant**:
- With/without `@c.us` or `@lid` suffix
- `+20` vs `20` vs `0` prefix variants
- Compares user-part only (digits before `@`)

### Tests

10 new tests for `isInWhitelist` + `decideAvailability` with test mode scenarios. Total: **59/59 passing**.

## How to use (admin workflow)

**Soft launch protocol:**
1. Open AI Agent tab in CLARK
2. Toggle "تشغيل" (master) → green pulse
3. Schedule tab → "وضع التجربة" section → toggle ON
4. Add your phone first (so YOU can test from another device)
5. Add 3-5 friendly customer numbers
6. Save Changes
7. Send a test message from a whitelisted number → reply
8. Send from an outside number → "تحت الاختبار" (or silent)
9. Iterate on the agent's behavior, FAQs, system prompt
10. When confident, toggle test mode OFF → live for everyone

## Cost protection rationale

The `clark-rmg.duckdns.org` WhatsApp number could be shared (intentionally or by mistake) before launch. Without test mode, every random sender triggers an Anthropic call (~$0.005-0.01). With test mode + whitelist:
- Whitelisted: full LLM call
- Non-whitelisted: free canned reply via bridge (no Anthropic)

This means the admin can keep the agent online during the entire soft-launch period without watching every message.
