# V19.76.0 — Product Catalog (single source of truth for AI Agent)

**Date:** 2026-05-06
**Type:** Feature

## ملخص

User report: *"سالته على كود موديل موجود بالفعل وقاللي انه الكود مش صح والموديل ده مش موجود... ازاي ندربه على الداتا"*

The agent was hallucinating about products because it had no curated catalog — it scanned `seasons/{s}/orders/` which has unstructured model data. Now there's a proper Catalog tab where admin curates products, and the agent reads it as **the** source of truth.

## What's new

### New tab: 📦 الكتالوج (between الشخصية and الأسئلة المتكررة)

**List view:**
- Responsive card grid (auto-fill 280px wide)
- Each card: 4:3 product image, monospace code, name, season/category badges, description, sizes/colors summary, price + min order qty
- Search bar + category filter + season filter
- Stats pills: total / filtered / in-stock
- Warning at 50+ items (Firestore doc size approaching limit)

**Editor modal:**
- Code + name (required) + English name (optional)
- Category dropdown (ولادي/بناتي/بيبي/junior/أخرى)
- Season dropdown (from `data.seasons`)
- inStock toggle
- Description textarea
- Image upload (auto-compressed to 4:3 / 600px / ~30KB JPG via existing `compressImg43`)
- Sizes + colors tag editors
- Fabrics multi-select (from `data.fabrics`)
- Wholesale price + min order qty
- Internal tags + notes

**Import from orders:**
- Click "📥 استيراد" → scans `data.orders[]` for unique `{code, name}` not yet in catalog
- Sorts by frequency (most-ordered first)
- Bulk-select + import — instant catalog seed from existing data

### Schema additions

```js
// constants/index.js — INIT_CONFIG
catalog: [
  {
    id, code, name, nameEn,
    image,            // compressed base64 thumbnail
    description, category, season,
    sizes: [], colors: [], fabrics: [],
    priceWholesale, minOrderQty,
    inStock, notes,
    tags: [],
    createdAt, updatedAt,
  },
],
catalogCategories: ["ولادي", "بناتي", "بيبي", "junior", "أخرى"],
```

### Backend changes (clark-ai-agent)

**`searchProducts` rewrite:**
- No more scanning the orders subcollection
- Reads `config.catalog[]` directly
- Scoring: code exact=1000, code substring=100, name exact=800, name substring=80, English name=60, tag exact=50, tag substring=20, description=30
- Returns structured data without image data (LLM context efficiency); flag `has_image` for Phase 2 media replies
- Returns explicit anti-hallucination message when no match: *"ما لقيتش موديل ... لا تخترع — لو العميل بـ يصرّ، استخدم escalate_to_human."*

**New tool: `get_product_details(code)`:**
- Exact code lookup
- Returns full product info or anti-hallucination message
- Used after `search_products` narrows down

**System prompt — catalog-aware factory facts:**
- Catalog count + in-stock count
- Top 4 categories with counts
- 6 sample model codes (`code/name`) as concrete examples for the LLM
- Empty-catalog explicit instruction: *"لا تخترع — استخدم escalate"*

### Anti-hallucination guardrails

3 layers reinforce the same principle:
1. **System prompt facts:** "لا تخترع موديلات. الكتالوج فيه X موديل، أمثلة Y..."
2. **Tool description:** "⚠️ لو search_products بـ يرجع 0، الموديل مش موجود — لا تخترع."
3. **Tool result message:** Explicit anti-hallucination text in the JSON returned to the LLM.

## How to use (admin workflow)

1. **First time:** Open AI Agent → 📦 الكتالوج → click "📥 استيراد" → bulk-add discovered models from orders
2. **Edit each:** Click ✏️ on a card → set image, category, sizes, colors, fabrics, price, min order qty
3. **Add new:** Click "+ موديل" → fill the form → 💾 حفظ
4. **Save Changes** at the page header pushes everything to Firestore
5. **Test:** Ask the agent about a model code in the Sandbox tab or via WhatsApp

The agent will now:
- Answer correctly when asked about a model in the catalog
- Refuse to invent prices/specs for unknown codes (escalates instead)
- Use real data: "موديل WINTER PRO (3262111)، السعر 320 ج، الحد الأدنى 50 قطعة، متاح بالألوان أزرق وأحمر، مقاسات 2-12"
