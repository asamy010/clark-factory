# V19.70.7 — Hotfix: checkPaymentReceived missing from Triggers UI

**Date:** 2026-05-05
**Type:** fix, ux
**Status:** shipped

## Critical fix

V19.70.5 added `checkPaymentReceived` to:
- `EVENT_VARIABLES` in `eventBuilder.js` ✓
- `DEFAULT_AUTOMATION_CONFIG.events` in `buildDailyReport.js` ✓

But missed the **third** required registration: the `eventTypes` array in `AutomationPg.jsx` TriggersTab. Without it, the user couldn't enable the trigger from the UI — the client-side hook would call `/api/event-trigger`, the endpoint would respond `skipped: event-disabled`, and no message would fire.

**Fix:** Added `'checkPaymentReceived'` to the array (now 5 events).

## UX improvement: Arabic-primary labels

Old labels mixed English/Arabic:
- `'💰 Sale Completed'`, `'💵 Payment Received'`, etc.

User feedback was that the cash and check flows needed clearer separation — "الخزنة لوحدها والشيكات لوحدها." New labels are explicit:

- `💰 بيع جديد للعميل`
- `💵 دفعة كاش/تحويل من عميل` (نقدي/محفظة/انستاباي/تحويل بنكي — مش شيكات)
- `🏦 دفعة شيكات من عميل` (شيك واحد أو حافظة)
- `⚠️ أوردر متأخر`
- `📅 شيك يستحق قريباً`

## Test plan for regression

Any new event type MUST be registered in 3 places:
1. `EVENT_VARIABLES` in `src/utils/automation/eventBuilder.js` (also mirror in `api/_eventBuilder.js`)
2. `DEFAULT_AUTOMATION_CONFIG.events` in `src/utils/automation/buildDailyReport.js` (mirror in `api/_buildDailyReport.js`)
3. `eventTypes` array in `src/pages/AutomationPg.jsx` TriggersTab

Missing any one of them ghosts the event from the user's perspective.
