## Architecture decision: CLARK = config UI, Agent backend = separate project

| Component | Where | Status |
|-----------|-------|--------|
| AI Agent backend (Node.js + Claude + bridge) | Contabo VPS, separate repo `clark-ai-agent` | Phase D (not built) |
| AI Agent admin UI | Inside CLARK app (this version) | Phase A done |
| Bridge (WhatsApp gateway) | Contabo VPS (existing) | Already running |
| Config + logs storage | Firestore (shared) | aiAgent collections (TBD) |

**Why config-first:** الـ admin يقدر يـtrain الـ Agent من دلوقتي (FAQs + system prompt). لما الـ backend يبني، هيقرأ نفس الـ config — مفيش re-work.

**Read-only constraint** (per spec): الـ Agent (لما يطلع) ميـكتبش على CLARK collections. كل الـ writes بتاعته على `aiAgent*` collections منفصلة. ده security boundary بيتـenforce في الـ backend's Firestore wrapper. CLARK app هي اللي بتعدّل الـ config — هي مش الـ Agent.
