## 🐛 المشكلة المُبلَّغ عنها (بعد V19.55)

**المستخدم**: "رحلت 5 فواتير تمام، بس ظهر toast 'فشل ترحيل الفواتير'. لما رحلت 80 فاتورة، اختفوا فوراً، بعد 5 ثواني رجعوا تاني ناقصين واحدة، وكل ثانيتين بيرحل واحدة أو اتنين تدريجياً."

## 🔍 التشخيص الكامل

V19.55 (الـserialization) كان صح من ناحية **correctness** — الـwrites أخيراً بتوصل في الترتيب الصح. لكن كشف 2 مشاكل في الـUX:

### مشكلة 1: progress modal كاذب
```
runBulkPost loop:
  for(item) {
    await postOne(item)   ← ينتهي بعد الـoptimistic update فقط
                            (الـsetDoc لسه في الـqueue!)
    setProgress(done++)   ← progress يقول "done" قبل الفعلي
  }
  setProgress(finished)   ← "تم" يظهر بعد ثانية واحدة
  
الواقع:
  Queue contains 80 × ~5 setDocs = 400 writes
  Sequential @ 100-300ms each = 40-120 seconds
```

السبب الجذري: `upConfig` كان **fire-and-forget**. بيـreturn `undefined` بدل ما يرجع الـpromise بتاعة upConfigTx. الـcallers مش قادرين يعرفوا امتى الكتابة خلصت فعلاً.

### مشكلة 2: false 'فشل' toast
في handlePost:
```js
const postPromise = autoPost.salesInvoicePosted(...)
  .then(res => { upConfig(d => ...) })
  .catch(e => console.warn(...));   ← بيبلع الـerror
return postPromise;
```

- لو autoPost نجح → promise resolves → loop يعدّ success
- لو autoPost فشل → `.catch` يبلع → promise resolves → loop يعدّ success كذلك (غلط!)
- لكن لو exception طلع **قبل** الـ.catch (مثلاً customer lookup throws) → loop يعدّ fail
- النتيجة: counter غلط → toast غلط

## ✅ الإصلاح في V19.56

### 1. `upConfig` / `upSales` / `upTasks` بيرجعوا الـTx promise

```js
// قبل V19.56
upConfigTx(next, ...);   // لا return → caller يستلم undefined

// بعد V19.56
return upConfigTx(next, ...);   // ← caller يستلم Promise
```

دلوقتي `await upConfig(...)` يستنى **الـactual setDoc** يخلص (مع الـqueue serialization).

### 2. `handlePost` في كل صفحات الترحيل اتعمل refactor

```js
// قبل V19.56 (Sales/Purchase/CreditNotes/DebitNotes)
upConfig(d => mark posted);  // fire-and-forget
const postPromise = autoPost(...).then(res => upConfig(...)).catch(...);
return postPromise;          // resolves AFTER autoPost only

// بعد V19.56
try {
  await upConfig(d => mark posted);              // wait write 1
  const res = await autoPost(...);                // wait autoPost
  if(res.ok) await upConfig(d => set ref);        // wait write 2
} catch(e) {
  throw e;                                         // surface to bulk loop
}
```

### 3. النتيجة

**Bulk loop**:
```js
for(const item of items) {
  await postOne(item);   ← دلوقتي ينتهي بعد كل الـwrites flush فعلاً
  setProgress(done++);   ← progress accurate
}
```

**Progress UI**: 
- 80 فاتورة هتاخد ~3 دقايق (بمعدل 1-3 فاتورة/ثانية)
- المستخدم يشوف عداد real-time
- لما ينتهي = فعلاً انتهى، مفيش "بعد 5 ثواني يرجعوا"

**Toast**:
- success → "✓ تم ترحيل N"
- partial fail → "⚠ X نجح، Y فشل"
- complete fail → "✕ فشل ترحيل N"
- counters صح لأن exceptions بتـpropagate صح
