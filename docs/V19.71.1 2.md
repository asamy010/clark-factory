# V19.71.1 — Hotfix: AI Agent page crash (React #306)

**Date:** 2026-05-06
**Type:** Fix (hotfix on top of V19.71.0)

## Symptom

User opens the new "AI Agent" tab → page renders the global `ChunkErrorBoundary` fallback ("حدث خطأ غير متوقع"). Console:

```
Minified React error #306; visit https://reactjs.org/docs/error-decoder.html
?invariant=306&args[]=undefined&args[]=
    at Cx (vendor-recharts-...js)
    ...
Chunk load error: Error: Minified React error #306
```

React error **#306** = "Element type is invalid: expected a string ... but got: `undefined`."

## Root cause

`src/utils/lazyLoad.jsx` exports `lazyNamed(importFn, exportName)` which expects a **named export** from the imported module:

```js
return lazy(()=>retryImport(importFn).then(module=>({
  default: module[exportName]  // module.AIAgentPg
})));
```

`App.jsx` calls `lazyNamed(() => import("./pages/AIAgentPg.jsx"), "AIAgentPg")` — so it reads `module.AIAgentPg`.

But V19.71.0's `AIAgentPg.jsx` was written with:

```js
export default function AIAgentPg(...){ ... }
```

So `module.AIAgentPg` is `undefined`. React.lazy receives `{default: undefined}`. Suspense resolves and React tries to render `<undefined ... />` → throws error #306. The error surfaces in the recharts vendor chunk because that's where the active Suspense boundary lives at the time of the failure (DashPg chart code).

## Fix

One-character change in `src/pages/AIAgentPg.jsx`:

```diff
- export default function AIAgentPg({ data, upConfig, isMob, canEdit, user }){
+ export function AIAgentPg({ data, upConfig, isMob, canEdit, user }){
```

All other CLARK pages (HRPg, CampaignsPg, AutomationPg, etc.) use named exports per the convention documented in `lazyLoad.jsx`'s header comment ("all CLARK pages use named exports"). I should have followed it from the start.
