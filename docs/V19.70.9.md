**Date:** 2026-05-05

## Searchable picker for العميل/المورد in check form

The native `<select>` rendered a long list of customers/suppliers — hard to navigate when there are dozens.

**Fix:** replaced with a searchable input + filtered dropdown:
- User types part of the name → list filters case-insensitively → click on a result links the `id` + `name`.
- For 30+ results, the dropdown caps at 30 and shows a count for the rest (`'...N نتيجة أكتر — اكتب أكتر للتضييق'`).
- When a party is selected, it shows with `✕` to clear and return to the search view.
- The manual-name fallback (for parties not in the registry) remains as a secondary input below.

## Centered time alignment under date in checks list

V19.70.8 added time under date but aligned start (right in RTL). User wanted the timestamp centered.

**Fix:** `textAlign: 'center'` on the `<td>` and the two divs. The timestamp is now in the central column under the date — visually clearer.
