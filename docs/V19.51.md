## 🐛 V19.51 HOTFIX: Customer Portal payments missing

**المشكلة المُبلَّغ عنها:** كشف العميل في الـportal مش بيعرض الدفعات (مع إنها ظاهرة في الخزنة وكشف العميل الداخلي).

**السبب:** الـserverless API endpoints (`api/customer-portal.js` + `api/workshop-portal.js`) كانت بتقرا `config.custPayments` و `config.checks` و `config.wsPayments` مباشرة من factory/config. لكن بعد migration V19.49، الحقول دي اتنقلت لـ`custPaymentsDays/`، `checksDays/`، `wsPaymentsDays/` collections — فالـAPI كان بيرجع arrays فاضية.

**ليه ما اتلقطش في V19.49 الأول؟** الـ Vite build بيختبر الـclient code بس (React). الـserverless functions في `api/*.js` بتشتغل على Vercel runtime منفصلاً ولا الـbuild يلمسها. كانت لازم اختبار manual للـportal بعد deploy.

**الإصلاح:**
- ضافت helper جديد في `api/_firebase.js`: `readSplitCollection(collectionName)` — يقرأ كل day docs ويجمعهم في array واحد عبر admin SDK
- `api/customer-portal.js`: بدل `config.custPayments` و `config.checks`، بقت تقرا من `custPaymentsDays/*` و `checksDays/*` لو الـflag `_splitDaysV1949Done` موجود (وإلا fallback على config للـbackward compat)
- `api/workshop-portal.js`: نفس الإصلاح لـ`wsPaymentsDays/*`

**Files modified في الـhotfix:**
- `api/_firebase.js` — added `readSplitCollection` export
- `api/customer-portal.js` — read custPayments + checks from day collections
- `api/workshop-portal.js` — read wsPayments from day collections

**Backward compat:** لو الـconfig لسه فيه الـflag undefined (نسخة قديمة قبل V19.49) → الـAPI بيقرا من config مباشرة. مفيش break لـdeployments قديمة.

**درس مستفاد للـV19.52+:** أي migration يحرك حقل من `factory/config` لـday collection لازم نـaudit كل الـAPI endpoints في `/api/` للحقل ده ونحدّثهم. الـclient build مش هيـcatch الـbug ده.
