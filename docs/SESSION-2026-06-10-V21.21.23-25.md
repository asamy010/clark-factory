## بروتوكول العمل المعتمد في الجلسة (ساري لكل التعديلات الجاية)

بعد **كل** تعديل، يتنفّذ التسلسل ده **أوتوماتيك من غير سؤال**:

1. **Build** — `npm run build` لازم ينتهي بـ `✓ built` وصفر أخطاء.
2. **Bump version** في ٣ أماكن:
   - `package.json` → `"version"`
   - `src/constants/index.js` → `APP_VERSION`
   - `src/components/AboutVersionModal.jsx` → entry جديد على رأس `CHANGELOG`
3. **Commit** — staging للملفات المحددة بس (مفيش `git add .`)، برسالة V-tagged.
4. **Push**:
   - فرع التطوير: `claude/code-review-3e12c`
   - النشر (production / Vercel): `main` (بـ fast-forward من فرع التطوير)
5. **Zip** — يتعمل ملف `clark-v<النسخة>.zip` (المجلد الداخلي = اسم النسخة،
   من غير `node_modules`/`dist`) ويتبعت للمستخدم.

> ملاحظة بيئة: الجلسة دي بتشتغل في بيئة سحابية (مش الماك). الزيب بيتعمل بـ
> `git archive` (الملفات المتتبّعة في git بس). أمر الـ rsync+zip بتاع الماك
> في CLAUDE.md §1 لسه صالح للتشغيل المحلي على الماك بعد `git pull`.

## V21.21.23 — إصلاح ظهور رقم التليفون مقلوب في التقارير والطباعة

**النوع:** fix · **التاريخ:** 2026-06-10

### المشكلة
رقم التليفون كان بيظهر مقلوب في تقارير وطباعات كتير، مثال:
`+20 12 12 05225091` بدل `+201008879265`.

### السبب الجذري (Root Cause)
في السياق العربي (RTL) خوارزمية الـ Unicode bidirectional algorithm بتعيد
ترتيب علامة الـ `+` وقطع الأرقام (runs) لما الرقم يكون فيه مسافات/فواصل أو
يكون جنب نص عربي. النتيجة: الرقم بيتقسم لـ runs وتتعكس بصرياً.

### الحل
helper مركزي جديد في `src/utils/format.js`:

```js
export function ltrPhone(p){
  const s = String(p == null ? "" : p).trim();
  return s ? "⁦" + s + "⁩" : "";  // U+2066 LRI … U+2069 PDI
}
```

بيغلّف الرقم بعلامات عزل اتجاهية (LTR isolate) فيتعامل معاه كوحدة واحدة LTR
مهما كان السياق. يشتغل في HTML الطباعة وفي JSX على الشاشة. **تغيير عرض بحت** —
حروف غير مرئية، مفيش أي logic اتمس.

## V21.21.24 — زر «مرتجع الكل» في مرتجع المشتريات

**النوع:** feature · **التاريخ:** 2026-06-10

### الطلب
في مودال مرتجع المشتريات (اللي بيفتح من الاستلام/الفاتورة)، زر يملأ كمية
المرتجع لكل بند بقيمة «المتبقي» بتاعته — لإرجاع كامل المتبقي أو كامل الاستلام
دفعة واحدة بدل الكتابة بند بند.

### التنفيذ
الملف: `src/pages/PurchasePg.jsx` — في footer مودال المرتجع (`returnRcpt`):
- زر **«↪️ مرتجع الكل (المتبقي)»**:
  ```js
  onClick={()=>setRetQty(Object.fromEntries(rows.map(x=>[x.i,x.rem])))}
  ```
  حيث `x.rem = المستلم − المرتجع السابق` (محدود بالحد الأقصى لكل بند).
- زر **«تصفير»**: `onClick={()=>setRetQty({})}` (بيظهر لما يكون فيه كمية محدّدة).
- الزرّين بيظهروا بس لما `anyAvail` (فيه كمية متبقية فعلاً).

**تغيير UI بحت** — مفيش أي مساس بمنطق الحفظ/المخزن/إشعار المدين
(`saveReceiptReturn`).

## V21.21.25 — طرق دفع قابلة للإضافة + صف فلاتر كشف الحساب أوفر للمساحة

**النوع:** feature + improvement · **التاريخ:** 2026-06-10

### الجزء (١) — صف فلاتر كشف الحساب على صف واحد
الملف: `src/components/AccountStatementView.jsx`

قبل: المورد/العميل في صف، و«من/إلى/رقم الفاتورة» في صف grid تاني.
بعد: الأربعة في **grid واحد**:
- ديسكتوب: `gridTemplateColumns: "2fr 1fr 1fr 1fr"` (الطرف بياخد عرض أكبر).
- موبايل: `"1fr 1fr"` (الطرف بياخد العرض كامل `gridColumn:"1 / -1"`).
- لو `fixedPartyId != null` (الطرف ثابت) → ٣ أعمدة بس.

### الجزء (٢) — طرق دفع قابلة للإضافة
الملف: `src/pages/TreasuryPg.jsx`

كانت «طريقة الدفع» قائمة ثابتة بـ ٤ خيارات hardcoded. دلوقتي:

1. ثابت module-level للخيارات الافتراضية:
   ```js
   const DEFAULT_PAY_METHODS = [
     { value: "نقدي كاش", emoji: "💵" },
     { value: "تحويل محفظة الكترونية", emoji: "📱" },
     { value: "تحويل انستاباي", emoji: "🔄" },
     { value: "تحويل بنكي", emoji: "🏦" },
   ];
   ```
2. القائمة المعروضة = الافتراضية + `data.payMethods` (المخصّصة) + القيمة
   المختارة حالياً لو مش في القائمة (عشان تعديل حركة قديمة مايضيّعش طريقتها).
3. زر **«➕ إضافة»** جنب الـ label → `askInput` لاسم الطريقة الجديدة (validate
   ضد الفاضي والمكرر) → بيتحفظ في `cfg.payMethods` عبر `upConfig`:
   ```js
   upConfig(d=>{ if(!Array.isArray(d.payMethods)) d.payMethods=[];
                 if(!d.payMethods.includes(t)) d.payMethods.push(t); });
   ```
4. الزر متاح بس لو `canEdit`.

### التخزين (قرار معماري)
طرق الدفع المخصّصة في `cfg.payMethods` — **قائمة إعدادات** (تعديل نادر، من
جهاز واحد غالباً) → `cfg` مناسب حسب قاعدة التخزين في CLAUDE.md §10
(«Settings only → cfg is fine»). **مش** محتاجة daily-split ولا per-id
partition.

### الظهور في كشف الحساب (كان شغّال أصلاً)
الطريقة المختارة بتتخزّن في `custPayments[].method` / `supplierPayments[].method`
وبتظهر في البيان في `src/utils/accounting/statement.js`:
- عميل: `"دفعة " + (p.method || "نقدي")`
- مورد: `"دفعة للمورد " + (p.method ? "(" + p.method + ")" : "")`

فاللي اتضاف فعلياً هو **إمكانية إضافة طرق جديدة**؛ الظهور كان موجود.
