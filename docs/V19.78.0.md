## What shipped

### 1. Stage classifier (`src/agent/stageClassifier.js`)

Heuristic 7-stage assignment computed at profile-load time:

| Stage | Trigger |
|---|---|
| Stranger | Phone not in customers |
| Awareness | Customer registered, 0 orders |
| Decision | 1 distinct order, recent |
| Customer | 2-5 active-season orders |
| Repeat | 6+ active-season orders |
| Dormant | Last delivery > 90 days |

Admin-set `customer.stage` (e.g. "VIP") wins over the heuristic.

The stage feeds into the system prompt's context block, so the agent's tone
adapts (Repeat → priority handling, Dormant → re-engagement attempt, etc.).

### 2. Tier helper (`src/agent/tierHelper.js`)

Auto-classifies from gross purchases (deliveries × sellPrice − returns)
in the active season.

| Tier | Default threshold |
|---|---|
| Bronze | 0 - 50K |
| Silver | 50K - 200K |
| Gold | 200K - 500K |
| Platinum | 500K+ |

Admin can override per-factory in `config.aiAgent.tiers = {Bronze:0, Silver:..., ...}`.

The context block now shows: `Tier: Gold (320K ج.م إجمالي مشتريات الموسم)`.

### 3. OTP system

Two new tools backed by Redis:

**send_otp**
- Generates 6-digit code, stores `otp:{request_id}` with 5min TTL (configurable)
- Sends via WhatsApp: "🔐 كود التحقق: 123456"
- Audit log in `aiAgentOtps`
- Returns `request_id` for `verify_otp`

**verify_otp**
- Takes `request_id` + `code`
- Match → success, code deleted (one-time use)
- Mismatch → bumps attempt counter (3 max)
- Expired/exhausted → returns specific error reason
- Updates audit log

### 4. Portal links

**generate_portal_link** — uses HMAC-SHA256 with shared secret to mint a
URL that clark-factory will accept:

```
https://clark-factory.vercel.app/?p=c&i={custId}&t={ts}&s={sig}
```

Where `sig = HMAC("portal:v2:{custId}:{ts}", CUSTOMER_PORTAL_SECRET).slice(0,12).base64url`.
90-day expiry baked into the timestamp. PII strict — sender only.

**Setup required for production:** add `CUSTOMER_PORTAL_SECRET` to the agent's
`.env` (must match the value in clark-factory's Vercel env). Without it, the
tool returns a graceful "not configured" message rather than crashing.

**generate_statement_pdf** — delegates to portal link with `focus=statement`,
returns the URL plus a friendly "افتح الرابط واضغط طباعة".

### 5. Auto-FAQ suggester

`src/agent/faqSuggester.js` is hooked into the orchestrator. When a turn
ends with `escalate_to_human` in toolsUsed, the user's question is added
to `aiAgentSuggestions` with `kind: "faq_suggestion"`.

Dedupe: SHA256 hash of the normalized question is the doc ID. Within a
7-day window, repeat occurrences just bump the counter; outside the window,
a fresh row is created.

The admin reviews these in the "🔔 اقتراحات الـ AI" tab (V19.77.2).

### 6. Voice (Whisper)

`src/agent/voice.js` exposes `transcribeVoice({base64, mimeType, language})`.
Uses OpenAI Whisper-1 (`$0.006/min`, ar language hint).

Webhook flow:
1. `type === "voice"` → decode base64 → call Whisper
2. If success → pass transcript through orchestrator like text
3. If `OPENAI_API_KEY` not set → "الصوت مش مفعّل، اكتب الاستفسار"
4. If transcribe fails → "تعذّر فهم الرسالة الصوتية"

Memory stores the transcript prefixed with `[رسالة صوتية مفرّغة]` so the
context shows the format.

**Setup required:** add `OPENAI_API_KEY=sk-...` to the agent's `.env`.

### 7. Image (Claude Vision)

Webhook flow:
1. `type === "image"` → orchestrator builds multi-block content:
   ```js
   content: [
     { type: "image", source: { type: "base64", media_type, data } },
     { type: "text", text: caption },
   ]
   ```
2. Claude Haiku 3.5 supports vision natively — no model swap needed
3. Memory stores `[صورة من العميل] — {caption}` (no base64) so follow-up
   turns don't keep heavy bytes around.

Use case: customer sends a model photo → agent uses `search_products` to
find the matching catalog entry.

## Required setup before testing

For Phase 3 features to fully work, the agent's `.env` on VPS needs:

```bash
# Required for portal links + statement PDF tools
CUSTOMER_PORTAL_SECRET=<copy from clark-factory's Vercel env>

# Optional — without this, voice messages fall back to "اكتب الاستفسار"
OPENAI_API_KEY=sk-...

# Optional — defaults to https://clark-factory.vercel.app
CLARK_BASE_URL=https://clark-factory.vercel.app
```

To set them:
```bash
ssh root@77.237.235.160
cd /root/clark-ai-agent
nano .env  # add the three lines above
pm2 restart clark-ai-agent
```

## Test scenarios (Cairo Arabic)

### Stage / Tier
1. **"احكي بياناتي"** — agent sees stage + tier in context, mentions both naturally.

### OTP
2. **"احكي رصيدي"** (from stranger LID claiming to be customer X) — agent uses
   `notify_admin_phone_request` (existing) AND/OR `send_otp` to verify.
3. Customer sends back the 6-digit code → agent uses `verify_otp` → on success,
   reveals balance.

### Portal / Statement
4. **"ابعتلي رابط حسابي"** — agent uses `generate_portal_link`, customer gets
   the URL.
5. **"ابعتلي كشف حسابي PDF"** — agent uses `generate_statement_pdf`, returns
   the portal URL focused on statement.

### Voice
6. Customer sends a voice note "كم باقي عليّ؟" — agent transcribes via
   Whisper, then runs the orchestrator like text → `get_customer_balance`
   → reply.

### Image
7. Customer sends a photo of a model → caption "ده موجود؟" — agent uses
   Claude Vision to identify, then `search_products` for catalog lookup.

### Auto-FAQ
8. Customer asks something the agent can't answer → agent uses
   `escalate_to_human` → orchestrator silently writes a `faq_suggestion`
   to `aiAgentSuggestions`. Admin sees it in the suggestions tab and can
   add it to FAQs manually.
