## 1. Always-on inputs + local-state grid

### Before (the flicker bug)

```
User clicks cell  →  setEditCell(k)
User types "24"   →  setEditVal(24)
User blurs/Tab    →  saveCell(...) → upSales → Firestore RTT (50-200ms)
                  →  setEditCell(null)            ← cell exits edit mode IMMEDIATELY
                  →  Old value displays for ~150ms while RTT pending
                  →  Firestore listener → data updates → re-render → 24 displays
```

The flicker = the gap between `setEditCell(null)` (cell exits to read-only mode) and the Firestore round-trip completing. The user sees their typed number disappear and reappear.

### After

The cell IS the input. Always editable (when `sessCanEdit`). Bound to a local state object `localGrid` (or `localAudGrid` for the audit popup). Every keystroke updates local state — no Firestore. The user clicks a single "💾 حفظ التغييرات" button at the footer to commit everything in one upSales call.

```js
const [localGrid, setLocalGrid] = useState({});
const [localGridDirty, setLocalGridDirty] = useState(false);

useEffect(() => {
  if (!activeSession) { setLocalGrid({}); setLocalGridDirty(false); return; }
  const sess = (data.custDeliverySessions||[]).find(s => s.id === activeSession);
  setLocalGrid({ ...(sess?.grid || {}) });
  setLocalGridDirty(false);
}, [activeSession]);

// onChange — write to local
setLocalCellQty(m, c.id, v) → setLocalGrid(prev => ({ ...prev, [k]: v })) + setDirty(true)

// On save
saveAllLocalGrid(sessId) → upSales(d => { d.custDeliverySessions[i].grid = cleanedLocalGrid })
```

### Auto-save on close

Closing the popup via the X button or backdrop click auto-saves any unsaved edits. The user can't accidentally lose data by clicking outside.

## 2. Max-avail validation

For each cell `(group, customer)`:

```js
availForGroupCell(m, custId) {
  // Sum each sub-order's available capacity, excluding the current customer
  for each sub-order oid:
    subStock = sm.stockQty
    subSold = sm.custDel
    otherPlan = sum localGrid[oid+"_"+other_cust] for cust ≠ custId
    capacity += max(0, subStock - subSold - otherPlan)
  return capacity
}
```

If the cell's local qty exceeds `availForGroupCell`, the visual changes:
- Border turns red (`T.err`)
- Background gets a red tint (`T.err+"08"`)
- Text turns red
- A "⚠️ تخطى" badge appears below the input
- Tooltip shows: `المتاح: <N>`

The value is NOT blocked — the user can still type whatever they want. We just warn loudly. This matches the existing pattern in saveCell which warns ("تحذير فقط") rather than rejecting.

## 3. Dashboard card vs popup mismatch

### Before

```js
// Dashboard card
totalRemain = stockModels.reduce((s,m) => s + m.avail, 0)  // includes avail < 0

// Popup
totalAvail = stockModels.filter(m => m.avail > 0).reduce(...)
```

If some models are oversold (avail < 0), the card sums them in (subtracting). The popup excludes them. Result: card 666, popup 687, user confused.

### After

```js
// Dashboard card
totalRemain = stockModels.filter(m => m.avail > 0).reduce(...)
```

Both compute the same way: only positive availability. Oversold models still appear in the matrix with red indicators where appropriate.

## Known limitations / not changed

- The OCR feature in the audit grid (📸 button per customer) still uses the old `saveAuditCell` direct path because it commits multiple cells at once and is logically a server-side data import, not a user-typed entry. If the user wants to also defer those to the local-state pattern, V19.70.23 can wire it up.

- The Tab key currently navigates within a row (next column). Shift+Tab is not handled (browser default). If the user wants Enter to commit + move down a row, we can add that in a follow-up.
