# V19.70.10 — 3 new check events: Issued + Collected + Bounced

**Date:** 2026-05-05
**Type:** feature, ux
**Status:** shipped

## Events added

### `checkPaymentIssued` — شيكات أوراق دفع لمورد
Same UX pattern as `checkPaymentReceived` but in the opposite direction. When a check with `type=payable + category=دفعة مورد + supplier linked` is saved, the supplier receives an instant message with full check details (bank, number, value, due date) plus progressive balance (for batch shipments, our debt to the supplier decreases per check).

A new `'supplier'` recipient role was added to `buildEventMessages` — uses a separate `supplierPhone` parameter (alongside `customerPhone`).

### `checkCollected` — تم تحصيل شيك
When a receivable check is marked as `status='محصل'`, the customer receives an instant thank-you (تم التحصيل، البنك، الشيك، التاريخ، الرصيد المتبقي).

The client hook snapshots the check before `upConfig` and fires after the existing toast.

Idempotency key: `checkCollected:${id}` — non-repeatable, so no date suffix.

### `checkBounced` — شيك مرتد
When a receivable check is marked as `status='مرتد'`, the customer receives a warning (شيك مرتد، البنك، الشيك، تاريخ الارتداد، الرصيد المستحق + 'يرجى التواصل فوراً للسداد').

Idempotency key: `checkBounced:${id}` — initially non-suffixed, but V19.70.11 changed this to date-suffixed for re-bounce support.

The balance reflects the reversal (the check is no longer counted as paid).

## UX

- **Supplier role label in EventCard** — recipient toggles now show `🏪 المورد` for the supplier role, alongside `👤 العميل` and `🏭 المالك`. Each event shows only the recipient roles relevant to it.
- **`تسجيل` column header centered** — V19.70.9 centered the data; V19.70.10 finally centers the header to match.

## API surface

`event-trigger.js` and `_eventProcessor.js` both accept `supplierPhone` in the body alongside `customerPhone`. The internal `phones` object now has 4 keys: `customer / supplier / owner / salesperson`. Each event uses whichever ones it needs based on `recipientRoles` in `EVENT_VARIABLES`.
