## Issue #1: Matrix "تخطى" warning false-positive

### السياق

User report:
> "ليه هنا بيظهر تحذير تخطي مع ان الكمية مظبوطة الـ 48 دخلوا و الـ 48 خرجوا.
> المفروض التحذير يحصل لما يحاول المستخدم يدخل كمية اكبر من الـ 48 صح؟"

السيناريو:
- Distribution session بـ مكتبين، كل واحد عنده 24 قطعة (24+24=48)
- الـ stock = 48 (إجمالي توزيع: 48، رصيد توزيع: 0، مباع فعلي: 48)
- كل خلية بـ 24 طلعت بـ تحذير "⚠️ تخطى"

### السبب

`availForGroupCell(m, custId)` كان بيحسب الـ cap كده:
```js
cap = subStock − subSold − otherPlan
    = 48 − 48 − 24 = -24  (Math.max(0, ...) → 0)
```

لكن الـ 48 المسجلة كـ `subSold` (sm.custDel) **هي بالظبط** الـ deliveries اللي
جت من الـ session ده (24 لكل مكتب). يعني الـ planned cells في الـ grid (24+24)
**ممثلة بالفعل** في الـ subSold. الطرح في الـ formula كان يـ double-count.

النتيجة: cap = 0 لكل خلية، أي قيمة > 0 تطلع تحذير.

### الإصلاح

اطرح فقط الـ deliveries اللي **مش من الـ session الحالي** (out-of-session
sold). الـ in-session deliveries محسوبة بالفعل في الـ plan cells.

```js
const inSessionDelivered = (o.customerDeliveries || [])
  .filter(d => d.sessionId === activeSess.id)
  .reduce((s, d) => s + (Number(d.qty) || 0), 0);
const outOfSessionSold = Math.max(0, subSold - inSessionDelivered);
cap += Math.max(0, subStock - outOfSessionSold - otherPlan);
```

السيناريو بعد الإصلاح:
- subStock=48, subSold=48, inSessionDelivered=48, outOfSessionSold=0
- لخلية مكتب A: q=24, otherPlan=24 (مكتب B), cap = 48 - 0 - 24 = 24
- exceeds = 24 > 24 = **false** ✓

التحذير دلوقتي بـ يفير فقط لما المستخدم يكتب رقم > الـ 24 المتاحة فعلاً.

## Issue #2: Filters in "تسليم جديد" popup

### السياق

User request:
> "عاوز في شاشة اختيار ماتريكس التوزيعة فلتر موديل وفلتر عميل"

الـ popup كان بـ يعرض كل الموديلات المتاحة (stockModels) و كل العملاء كـ
checkboxes. لو عندك 50 موديل و 100 عميل، الاختيار صعب.

### الإصلاح

ضافت input فلتر جنب كل قسم في الـ popup:
- **فلتر موديل** — يبحث في `modelNo` و `modelDesc`
- **فلتر عميل** — يبحث في `name` و `phone`

State جديد:
```js
const [newSessModelFilter, setNewSessModelFilter] = useState("");
const [newSessCustFilter, setNewSessCustFilter] = useState("");
```

الـ filtering case-insensitive (toLowerCase). لو فيلتر مفيش ليه matches، رسالة
واضحة "لا يوجد موديل مطابق" / "لا يوجد عميل مطابق" بدل قائمة فاضية.

## Issue #3: Sales hub redesign — 2-column grid

### السياق

User request:
> "الجزء ده من شاشة المبيعات عيد تصميمة، كل جزء يكون في مستطيل بنفس الشكل
> ومكتوب العنوان فوق المستطيل، ويكونوا جنب بعض على صفين مش صف واحد عشان
> ماتصغرش الايقونات"

قبل: 4 sections (العملاء، التقارير والتحليل، المخزن والجرد، أدوات أخرى) كانوا
stacked vertical, كل واحد سطر full-width. بياخد مساحة عمودية كبيرة.

### الإصلاح

```js
{(() => {
  const cardStyle = { border, borderRadius:12, background, padding:12, ... };
  const titleStyle = { fontWeight:800, marginBottom:10, paddingBottom:8, borderBottom };
  const btnsGrid = { display:"grid", gridTemplateColumns: isMob ? "repeat(2,1fr)" : "repeat(auto-fill,minmax(110px,1fr))" };
  return <div style={{display:"grid", gridTemplateColumns: isMob ? "1fr" : "repeat(2,1fr)", gap:12}}>
    <div style={cardStyle}><div style={titleStyle}>👥 العملاء</div>...</div>
    <div style={cardStyle}><div style={titleStyle}>📊 التقارير والتحليل</div>...</div>
    <div style={cardStyle}><div style={titleStyle}>📦 المخزن والجرد</div>...</div>
    {showOtherTools && <div style={cardStyle}><div style={titleStyle}>🔧 أدوات أخرى</div>...</div>}
  </div>;
})()}
```

### النتيجة

- **Desktop**: 2 columns × 2 rows — 4 cards بنفس الشكل، أوسع ومنظم
- **Mobile**: 1 column — كل card تحت اللي قبله (نفس behavior القديم)
- **Icons**: نفس الـ minmax(110px, 1fr) — مفيش تصغير
- **Title**: على رأس كل card مع border-bottom (titleStyle)

الـ section الأول (إجراءات سريعة) سايب زي ما هو — primary buttons منفصلة فوق.
