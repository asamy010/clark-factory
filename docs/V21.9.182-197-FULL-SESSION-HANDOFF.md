- ❌ `window.open(url)` بعد `await` (popup blocker)

- ❌ External `fetch` في serverless function بدون `AbortController` + timeout
- ❌ `INFLIGHT_LOCK_MS < cron tick interval`
- ❌ Destructive migration بدون dry-run + backup + atomic
- ❌ Self-healing migration بـ pre-state flag فقط (يحتاج on-demand repair)

## 🛡️ Important Lessons من الـ Session

### 1. **Production / Local divergence**
لما الـ user يـ verify production state، رد عليه بدقة. لا تـ assume الـ deployed version بـ يـ match الـ HEAD.

### 2. **Portal critical path**
الـ portal تـ used بـ مئات العملاء يومياً. أي change فيه يحتاج:
- Test محلي (ما عندناش)
- OR staging deploy + verification
- OR ULTRA-careful invariant analysis

V21.9.193-196 شفّت بدون verification → broke portal → roll back.

### 3. **17 commits في session واحد = compound risk**
طبقاً لـ CLAUDE.md §0.1 V21.9.67-69 precedent، الـ batching هو الـ anti-pattern. الـ correct path:
- One feature → push → verify → next feature

### 4. **Invoice-based vs delivery-based aggregation**
Both have edge cases. الـ invoice approach more accurate لـ overrides لكن requires reliable invoice data. الـ portal بـ يقرأ من Firestore Admin SDK — قد يـ diverge عن client read patterns.

### 5. **Defer is OK**
لما الـ fix غير verifiable safely، الـ correct call هو DEFER، مش ship. Ahmed's call لـ rollback was correct.
