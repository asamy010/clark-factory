## What's new

### 🎨 New robot icon

Old: smiling face circle. New: solid-fill robot head (square + antenna + side ears + 2 round eyes), matching the uploaded reference image.

The Home tile renders SVGs with `fill="none" stroke="currentColor"` by default. To get the solid look:
- Body/ears/antenna-bulb: `fill="currentColor"` (overrides parent)
- Eyes: `fill="white" stroke="none"` (pop on tinted bg)

### 📊 Dashboard Tab

**Reads from** `data.aiAgentAnalytics[YYYY-MM-DD]` (empty until Phase D backend ships).

- Range selector: اليوم / ٧ أيام / ٣٠ يوم
- 8 KPI cards: محادثات، رسائل، تحويلات، تكلفة (USD)، إشعارات مبيعات، طلبات اتأكدت، رسائل صوتية، صور
- Recharts BarChart: محادثات + تحويلات per day
- Cost breakdown: Anthropic (Claude) / OpenAI (Whisper) / Claude Vision
- Quick metrics: عملاء مختلفين، متوسط زمن الرد، رضا العميل (CSAT)، معدل التحويل، معدل تأكيد الطلبات
- Empty-state banner explaining "الـ backend Phase D لما يبني هيـكتب الإحصائيات هنا"

### 💬 Conversation Logs Tab

**Reads from** `data.aiAgentConversations` (empty until Phase D).

- Filter bar: text search + status dropdown (resolved/escalated)
- 3 stat pills: إجمالي / ظاهر / حُوّل
- Conversation cards with click-to-expand (full message thread)
- Empty state with schema preview: messages, tools_used, cost, escalation reason, CSAT

### 🧪 Sandbox Tab

**Local-only simulation. Does NOT call Anthropic.** Used for testing FAQs phrasings before backend ships.

- Setup: persona dropdown (known customer / stranger), customer picker from `data.customers`
- Chat UI with input field + send + 6 quick scenarios:
  - "كم سعر موديل WINTER PRO؟"
  - "حسابي عامل إيه؟"
  - "محتاج 200 قطعة"
  - "الشحن بياخد كم يوم؟"
  - "عاوز أتكلم مع شخص"
  - "ما هي طرق الدفع؟"
- Mock response logic:
  1. Match user input against FAQs (title + phrasings, substring both ways)
  2. If match: return answer with variable substitution (`{customer_name}`, `{today}`, `{tier}`)
  3. If no match: canned response acknowledging it's a sandbox
- Trace panel: shows pipeline steps with mock timings (Authentication → Stage classification → FAQ lookup → Response built)

### 🛠 Tools Tab

**Edits** `config.aiAgent.tools` + `tierDiscounts` + `escalation`.

**3 tool categories:**
- 📖 READ-ONLY (6): get_customer_info, search_products, get_customer_balance, get_customer_orders, get_order_details, get_faq_answer
- 📄 GENERATE (2): generate_portal_link, generate_statement_pdf
- 🔔 NOTIFY (5): notify_sales_team ⭐, notify_admin_phone_request, escalate_to_human, send_otp, verify_otp

Each tool has an enable/disable toggle. Tools with config show extras when enabled:
- `search_products`: includePricing, includeStock, includeImages, maxResults
- `generate_portal_link`: ttlHours
- `generate_statement_pdf`: businessHoursOnly
- `notify_sales_team`: maxValueBeforeManual
- `notify_admin_phone_request`: requiresOtp
- `send_otp`: ttlMin, maxAttempts

**Tier discounts editor:** Bronze 0% / Silver 3% / Gold 5% / Platinum 8% — inline number inputs.

**Escalation routing:**
- Support phone (single human)
- Sales team phone (group/individual)
- Template textarea with variables: `{customerName}`, `{phone}`, `{tier}`, `{stage}`, `{reason}`
- 5 auto-trigger toggles: shorty complaint / order > 100K / angry customer / out-of-scope / Platinum customer

### TABS metadata cleanup

Tabs with shipped functionality no longer have `phase` property → no badge shown. Phase C tabs (Funnel, Profiles) keep `phase: "C"` badge. Conditional simplified: `t.phase !== "A"` → `t.phase`.

## What's NOT live (yet)

- The agent doesn't actually call Claude or send WhatsApp messages. Sandbox is local-only.
- Dashboard + Logs read from Firestore collections that don't exist yet — empty states are expected.
- Tools tab config is saved to Firestore but no consumer reads it yet (Phase D).

## Sandbox is the most useful tab right now

Because the backend doesn't exist, the **Sandbox tab is where the admin can actually validate work**. They can:
1. Add FAQs (Personality + FAQs tabs)
2. Test if FAQs trigger on real-sounding user messages (Sandbox tab)
3. Iterate on phrasings until matches feel right
4. When backend ships, the same FAQs will be live

This is the value of building config UI before backend.
