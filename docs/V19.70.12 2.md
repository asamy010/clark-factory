# V19.70.12 — WhatsApp delivery receipts: bulk send PDF + details

**Date:** 2026-05-06
**Type:** feature
**Status:** shipped

## Per-customer PDF receipt + WhatsApp send

In the "طباعة مجمعة" popup in CustDeliverPg, a new button `📤 إرسال واتساب` appears next to the print button. For each selected customer with a phone:

1. HTML for the receipt is built (same layout as the print, but per-customer).
2. `html2canvas` + `jsPDF` convert it to PDF base64 (lazy-loaded from CDN — ~200KB, no bundle bloat for users who never trigger it).
3. A text message summary (تاريخ، إجمالي قطع، إجمالي قبل الخصم، الخصم، الصافي المستحق) is built.
4. POST to bridge `/send` with the PDF as media attachment. The bridge has supported this pattern since V19.31.

## Status badges per customer

Each customer in the list shows a state badge while sending:
- `⏳ جاري الإرسال` (transient)
- `✓ تم الإرسال` (success)
- `⛔ فشل` (with hover tooltip carrying the error message)
- `📵 بدون رقم` (for customers without a phone)

The send button locks during sending. The popup keeps the customers in the list after send (doesn't dismiss) — the badges show what landed.

## Anti-ban: bridge defaults

The send loop is sequential (`await` per customer, NOT `Promise.all`) — the bridge applies its default delays (8-15 seconds between messages) to avoid getting the WhatsApp number banned. For 50 customers this takes 6-12 minutes. The progress badge updates as each customer finishes. The confirmation prompt warns the user before the batch.

## New utility: `src/utils/htmlToPdf.js`

Reusable helper:
- `loadPdfLibs()` — lazy-loads html2canvas + jsPDF via CDN (caches in window).
- `htmlToPdfBase64(html, opts)` — renders HTML in an offscreen container, captures with html2canvas (`scale=2` for crisp text), builds a PDF (A4 portrait, multi-page automatic for tall captures), returns a base64 string ready for the bridge.

Reusable for any feature that needs HTML → PDF → WhatsApp.

## Failure handling

If one customer fails (network, bridge unreachable, PDF generation failed), the batch continues for the rest. The error is saved in `groupPrint.waLastErr[custId]` and surfaces as a hover tooltip on the failed badge. The send button stays enabled after the batch — the user can retry the failed ones (the successful ones have `✓` and won't re-send because the checkbox state holds the original selection).
