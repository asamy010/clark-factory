## Why V19.70.14 didn't work

V19.70.14 added `ensureCairoLoaded()` that:
1. Injected `<link>` to Google Fonts CSS
2. Called `document.fonts.load("800 12px Cairo")` for each weight
3. Awaited `document.fonts.ready`
4. 250ms safety delay

The user confirmed: **the fix didn't hold** — Arabic headers still rendered with disconnected glyphs in the PDF.

**Root cause:** `document.fonts.load(string)` is a **trigger**, not a **wait**. The API tells the browser "I will use this font, please load it" and returns a promise. But the promise can resolve before the actual TTF binary finishes downloading from Google Fonts. On a fresh page-load with cold cache, the binary is still in flight while the await already completes (~5-10ms). html2canvas then captures the offscreen container with Arial as the fallback — Arial doesn't shape Arabic ligatures, so each letter renders in its isolated form.

## Why V19.70.15 works

The `FontFace` API is the explicit, bulletproof way to load a font:

```js
const face = new FontFace("Cairo", "url(...)", { weight: "800" });
await face.load();         // Awaits the actual binary download + parse
document.fonts.add(face);  // Registers face document-wide
```

`face.load()` returns a promise that resolves **only after** the binary is fully downloaded and parsed. There's no race condition possible — the await blocks until the font is genuinely ready.

We also switched the source from Google Fonts CSS wrapper (`https://fonts.googleapis.com/css2?...`) to direct WOFF2 binaries from fontsource CDN (`https://cdn.jsdelivr.net/npm/@fontsource/cairo@5.0.13/files/cairo-arabic-{N}-normal.woff2`). Two benefits:
- The CDN serves the Arabic subset specifically (~30KB per weight vs ~200KB from Google Fonts that includes Latin extras)
- We can pass the URL directly to `new FontFace(...)` — no CSS parsing intermediate step

## Additional hardening

Beyond the font fix, V19.70.15 adds defensive layers around the html2canvas capture:

1. **Force reflow before capture** — `void container.offsetHeight` followed by double `requestAnimationFrame` ensures the browser has committed the font cascade for every node before html2canvas walks the DOM. Without this, the offscreen container's computed styles can be in a "pending" state.

2. **Image-load wait replaces 250ms timeout** — explicit await on every `<img>` element's `load` event (with 3s timeout cap). The 250ms timeout from V19.70.14 was a fragile hack that could miss slow CDN responses for the QR code image. Now we wait for the actual decode.

3. **Tighter font-family chain** — `font-family: Cairo, sans-serif` instead of `Cairo, 'Segoe UI', Arial, sans-serif`. If Cairo fails to load for any reason, the fallback is generic sans-serif (which the browser can pick to be a more reasonable Arabic-shaping font on most platforms). Arial specifically was the wrong fallback because it doesn't shape Arabic — silent breakage.

4. **Concurrent-call coalescing** — `_cairoLoading` promise caches in-flight loads. In a bulk send (50 customers), every customer's PDF generation calls `ensureCairoLoaded()` — they now all share the same single load promise instead of potentially racing.
