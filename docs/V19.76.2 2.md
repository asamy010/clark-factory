## السياق

User report:

> "رسالة واتس اب العميل الاوتوماتيك بالدفعة او الشيك بتجيب الرصيد بدون الخصم.
> العميل اخد بضاعة بـ1000 جنيه مع خصم 10% يبقى الرصيد على العميل 1000 قبل الخصم
> وبعد الخصم 900 وده الرصيد الفعلي اللي بنتعامل بيه دايماً.
> العميل دفع 200 جنيه دفعة. في الرسالة المفروض يخصم 900-200 ويقول الرصيد 700،
> انما هو بيحسب 1000-200=800 وبيقول الرصيد 800."

كشف الحساب في `CustDeliverPg` بيطبّق الـ customer discount % على إجمالي المبيعات
قبل ما يطرح الدفعات (`totalAfterDisc = totalVal − round(totalVal × disc/100)`)،
لكن الـ paymentReceived / checkPaymentReceived events ما كانوش بيطبّقوه — كانوا
بيحسبوا `Σ(deliveries × price) − Σ(returns × price) − Σ(payments)` فقط.

النتيجة: العميل يشوف رصيد في الـ statement مختلف عن اللي يوصله في الـ WhatsApp
message. لو الفرق هو الخصم بالظبط، العميل ييجي يقول "إيه الفرق؟" والمحاسب يـ
ضيع وقت يفسّر.

## الإصلاح

طبّقنا نفس الـ formula بتاعة كشف الحساب في 4 أماكن (cron + 3 client-side fires):

```
gross   = Σ(deliveries × price) − Σ(returns × price)
discAmt = Math.round(gross × discPct / 100)
balance = (gross − discAmt) − Σ(payments)
```

### 1. Cron — `api/automation-tick.js → computeCustomerBalances`

```diff

+ function computeCustomerBalances(orders, payments, customers){
    const grossByCust = {};
    const paidByCust = {};
    // ...accumulate gross + paid per customer...
+   const discPctById = {};
+   if (Array.isArray(customers)) {
+     for (const c of customers) {
+       if (c && c.id) discPctById[c.id] = Number(c.discount) || 0;
+     }
+   }
    const balances = {};
    for (const custId of allCustIds) {
      const gross = grossByCust[custId] || 0;
      const paid  = paidByCust[custId] || 0;
+     const discPct = discPctById[custId] || 0;

+     balances[custId] = (gross - discAmt) - paid;

  }
```
