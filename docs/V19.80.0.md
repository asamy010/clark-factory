## 2. OrdForm

### بحث الخامات (SearchSel)

قبل: `<Sel>` فيه list منسدلة بكل الخامات.
بعد: `<SearchSel>` مع `showAllOnFocus`. اكتب جزء من اسم الخامة → النتايج تـ filter (max 8). الـ keyboard navigation شغّال (↑↓ Enter).

### حقل قطع/راق لكل خامة

عمود جديد في صف الخامة:

```
[خامة A] [SearchSel] [استهلاك/راق] [Inp] [قطع/راق] [Inp] [تاريخ القص] [Inp]
```

**السلوك:**
- لما تـ enter قيمة (مثلاً 30) → الصفوف الموجودة في FCTable اللي قيمتها = الـ default القديم (size set's pcsPerSeries) أو 0 → بـ تـ auto-update للقيمة الجديدة، والـ qty يُعاد حسابه (`layers × pcsPerLayer`).
- الصفوف اللي عدّلها المستخدم يدوياً (قيمة مختلفة عن الـ default) ما تـ overwrite.
- الصفوف الجديدة بـ تستخدم القيمة الجديدة كـ default.
- الـ FCTable's pcsPerLayer Sel بـ يعرض multiples من القيمة (30, 60, 90, 120, 150).

### AccPicker — popup اتشال

قبل: زر "+ اختيار اكسسوارات" يفتح popup فيه checkbox grid.
بعد: حقل بحث inline. اكتب → النتايج تظهر تحت (max 12). كل نتيجة فيها:
- اسم البند
- الوحدة + السعر/قطعة
- 📦 المخزن المتاح (chip أخضر لو متاح، أصفر لو 0)

اضغط على الصف → بـ يضاف لجدول البنود تلقائياً + الـ search field بـ يـ clear.

## 6. ملاحظة على الـ Firestore 404

المستخدم لاحظ في الـ console:
```
firestore.googleapis.com/:1  Failed to load resource: the server responded with a status of 404
```

ده بـ يحدث لما `onSnapshot` listener يتأسس على document مش موجود. الـ Firebase SDK بـ يـ retry والـ listener بـ يـ work normally بعدها (يعرض "no document" event). مش error حقيقي بل debug noise من الـ network panel.

الـ "Could not establish connection. Receiving end does not exist." من Chrome extension (مش من الـ app). الـ "@firebase/firestore: enableIndexedDbPersistence() will be deprecated" warning من الـ SDK نفسه — في المستقبل هنـ migrate لـ `FirestoreSettings.cache`.

لا يوجد action item عاجل على الـ 404. الـ lazy loading للصور خفّف الـ "loading" feel اللي المستخدم لاحظه.
