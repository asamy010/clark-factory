# V19.70.21 — Approach A: jsPDF + Cairo TTF embedded (no html2canvas)

**Date:** 2026-05-06
**Type:** fix (architectural), feature
**Status:** shipped — pending user verification of the popup PDF

## Why this version exists

Four prior attempts at fixing the Arabic shaping bug in html2canvas-rendered PDFs (V19.70.14 link injection, V19.70.15 FontFace API, V19.70.16 system fonts, V19.70.19 `<td class='h'>`) failed. The user confirmed each attempt: headers still scrambled.

After exhausting font-loading and element-type workarounds, it became clear the bug is **structural** — html2canvas's internal canvas rendering of complex scripts (Arabic, Hebrew, Devanagari) is unreliable cross-browser. No CSS or font tweak fixes it.

The user-confirmed plan: **drop html2canvas from the PDF pipeline entirely**. Use jsPDF directly with a TTF font embedded in the PDF.

## What's in this version

A new utility `src/utils/arabicPdf.js` that generates PDFs without html2canvas. The pipeline:

1. **Lazy-load jsPDF + jspdf-autotable** from cdn.jsdelivr.net.
2. **Lazy-fetch Cairo TTF binaries** (Regular + Bold) from `cdn.jsdelivr.net/gh/google/fonts@main/ofl/cairo/static/` — CORS-safe and reliable. Convert each ArrayBuffer to base64 (chunked to avoid call-stack overflow).
3. **Register fonts with jsPDF** via `pdf.addFileToVFS('Cairo-Regular.ttf', base64)` + `pdf.addFont('Cairo-Regular.ttf', 'Cairo', 'normal')`. Same for Bold. The fonts are now embedded directly in every PDF this engine produces — no external dependency, no cache races.
4. **Set RTL mode** via `pdf.setR2L(true)`.
5. **Shape Arabic text** via the embedded shaper before passing to `pdf.text()` or autoTable cells. jsPDF doesn't auto-shape Arabic — it just renders whatever glyphs the codepoints map to. The shaper converts Unicode Arabic logical-order text (e.g. `العميل`) to its visual presentation form using the Arabic Presentation Forms-B block (U+FE70-U+FEFC).

### The Arabic shaper

~150 lines, implements the standard Arabic contextual joining algorithm:

- **Joining type per letter:**
  - Right-joining (joins TO previous, breaks chain to next): ا د ذ ر ز و ء ؤ ة ى
  - Dual-joining (joins both sides): most letters — ب ت ث ج ح خ س ش ص ض ط ظ ع غ ف ق ك ل م ن ه ي

- **Per-character form selection:**
  - `medial` if previous-joins-forward AND next-joins-backward
  - `final` if only previous-joins-forward
  - `initial` if only next-joins-backward
  - `isolated` otherwise

- **Lam-Alef ligature:** ل + ا (and variants ل + آ، ل + أ، ل + إ) collapse to a single glyph (U+FEFB / U+FEFC).

- **Non-Arabic characters:** pass through unchanged. Numbers, Latin, punctuation render normally.

- **Output order:** the shaper reverses the array at the end so the visual order matches RTL layout.

### What this version applies to

The V19.70.20 `رصيد متاح` popup PDF (the simpler case) is migrated:

- Old: `htmlToPdfBase64(buildReportHTML())` → html2canvas captures HTML → PDF
- New: `buildAvailableStockPdfBase64({factoryName, totalAvail, totalSeries, totalBroken, modelCount, rows, ...})` → jsPDF text APIs → PDF

The structured payload replaces the HTML build step entirely. Same visual layout (header + 4 chips + table + footer) but constructed via `pdf.text()`, `pdf.roundedRect()`, and `pdf.autoTable()`.

The bulk delivery receipt (`buildOneCustomerHTML` in CustDeliverPg) is still html2canvas-based. It's a more complex layout (header with logo, info table, items table, discount block, QR code image, signature row, footer) — substantial work to rebuild via jsPDF. **V19.70.22 will migrate it** if V19.70.21's popup PDF renders Arabic correctly. The user can still send via WhatsApp using the V19.70.17 text-only mode (the includePdf checkbox stays default OFF).

## If something doesn't render correctly

Most likely failure modes and fixes:

1. **Some letters appear as boxes (□) or wrong shape** → the Cairo font's cmap may not map the presentation form codepoint we generated. Cairo has full Arabic glyph coverage so this is unlikely, but verify by trying a different font (Amiri-Regular.ttf has a complete Arabic Presentation Forms-B block).

2. **Letters disconnected even though the form is shaped** → the shaper picked the wrong form. Possible cause: the joining-type table is incomplete for that letter. Add the missing letter to `_AR_FORMS` in `arabicPdf.js`.

3. **Whole text reversed (showing as mirror)** → the `out.reverse()` at the end of `ar()` is double-applied with jsPDF's `setR2L`. Try removing one of them.

4. **Font fails to load** → CDN issue with jsdelivr/gh. Switch to a different mirror or bundle the TTF as a static asset in `public/fonts/`.
