## السياق

V19.77.1 added the `notify_admin_phone_request` tool. When an LID-only sender
claims to be customer X, the agent writes a doc to `aiAgentSuggestions` with
`kind: "lid_phone_mapping"`. But there was NO UI to act on those suggestions —
they just piled up invisibly in Firestore.

V19.77.2 closes that loop with an admin review queue.

## What shipped

### New tab: 🔔 اقتراحات الـ AI

Added as the 8th tab (between Logs and Sandbox). Live subscription to
`aiAgentSuggestions` ordered by `sent_at` desc, limit 100.

**Header bar** (filter chips + counts):
- ⏳ تحت المراجعة (N)  ← default
- ✓ مربوط (N)
- ✗ تجاهل (N)
- 🚫 محظور (N)
- الكل (N)

**Per-card layout** (for `kind: "lid_phone_mapping"`):
- 🔒 LID badge if applicable + monospace WID
- 👤 الاسم المدّعى, 📞 الرقم المدّعى, 💬 reason (italic snippet)
- Status badge (color-coded)
- Action buttons (right side, only when pending + canEdit):
  - **✓ ربط بعميل** → opens inline picker
  - **✗ تجاهل** → marks `ignored`
  - **🚫 حظر** → marks `blocked`

**Link picker** (inline expand under the card):
- Search input (filters customers by name + phone)
- Radio list of customers (top 30, scroll to load search result subset)
- "✓ تأكيد الربط" appears only when a customer is selected

### Link decision flow

```js
// Step 1: write to customers via upConfig
upConfig(d => {
  const cust = d.customers.find(c => c.id === pickedCustomerId);
  cust.additional_phones.push({
    number: s.wid,
    added_via: "ai_suggestion",
    added_at: new Date().toISOString(),
    suggestion_id: s.id,
  });
});

// Step 2: mark suggestion as decided (direct Firestore write)
await setDoc(doc(db, "aiAgentSuggestions", s.id), {
  status: "linked",
  decision: "linked",
  linked_customer_id: pickedCustomerId,
  reviewed_at: new Date().toISOString(),
}, { merge: true });
```

The next time the same LID messages, the agent's `profileLoader` will see the
LID in `customer.additional_phones[].number` and resolve to the matched
customer instead of falling to Stranger flow.

### Live pending-count badge

Top-level subscription to `aiAgentSuggestions` lifts the pending count to the
parent component. The "🔔 اقتراحات الـ AI" tab shows a small red badge with
the count whenever > 0. Admins see at a glance if there's anything to review.

### Forward-looking design

`SuggestionCard` is generic — only the `lid_phone_mapping` kind has a
specific layout + link action. Other kinds (`faq_suggestion`,
`customer_observation`, `stage_transition`) fall back to a JSON dump until
they get specific UIs. The kind filter chips appear only when > 1 distinct
kind exists in the data.
