# V19.70.8 — Check batch progressive balance + UI polish + multi-select delete

**Date:** 2026-05-05
**Type:** fix, feature, ux
**Status:** shipped

## Critical fix: progressive balance for batch checks

User report: a حافظة of 3 checks was sending messages with the SAME balance for all 3 (`'1,600 ج.م'` for each). Should have been progressive — شيك 1 = base − 1000, شيك 2 = base − 2000, شيك 3 = base − 3000.

**Root cause:** the client hook computed balance once before the loop and reused it for every check.

**Fix:**
- Compute `baseBalance` (without any check from the batch).
- For each check `i`, send `baseBalance - (i+1) * amount`.
- Replace `Promise.all` with sequential `await` so the messages arrive in order 1→2→3 in WhatsApp (otherwise the bridge can interleave them based on network timing).

Same fix applied to the cron-side scan in `api/automation-tick.js`'s `scanRecentChecks`. For each check that is part of a batch, sum the amounts of earlier checks in the same `batchId` (using `batchIdx`) and subtract. For single (non-batch) checks, just subtract this check's amount. Client + cron now agree on balance logic.

## UX: time UNDER date in checks list

The cell used to display the time inline after the date. User wanted the time stacked under the date.

**Fix:** the cell now contains two clean divs:
```
<div>{c.date}</div>
<div>{formatTxTime(c.createdAt)}</div>
```
With `lineHeight: 1.3` and `padding: 4px` (down from `6px`) — the row compressed ~30%, more checks visible without scrolling.

## Feature: multi-select + bulk delete for checks

Before V19.70.8 the user had to delete checks one by one. Now:
- Checkbox column at the start.
- Select-all in the header.
- When 1+ checks are selected, a red bulk-delete bar appears above the table with two buttons: "إلغاء التحديد" and "حذف المحدد (N)" (with confirmation).

Same pattern as the journal's transaction multi-select. Especially useful when a batch was created by mistake and needs to be removed wholesale.
