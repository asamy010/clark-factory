# V19.70.26 — Amiri (full PFB) + shaper algo fix + QC-2 larger

**Date:** 2026-05-06
**Type:** fix, ux
**Status:** shipped

## What broke in V19.70.25

User reported: "الكتابة فيها حروف ناقصة كتير، يعني حرف الألف في كل كلمة مش موجود وحرف الراء في كلمة التاريخ، الألف والنون في كلمة التليفون".

Looking at concrete examples from the screenshot:
- `التليفون` rendered as `لتليفو` — missing ا (start) AND ن (end)
- `التاريخ` rendered as `لتايخ` — missing ا AND ر
- `العنوان` rendered as `لعنو` — missing ا AND ا AND ن
- `الموديل` rendered as `لمويل` — missing ا AND د
- `الإجمالي` rendered as `لإجملاي` — letters reordered + missing

This wasn't an order bug (the visible letters were in the correct visual order). It was a **missing-glyph** bug. Two contributing causes:

1. **Tajawal's cmap is incomplete for Arabic Presentation Forms-B (PFB) block (U+FE70–U+FEFC).** When the shaper emitted (e.g.) 0xFE8D for ا-isolated, 0xFEE5 for ن-isolated, 0xFEAE for ر-final, etc., Tajawal didn't have those glyphs — so jsPDF silently skipped them. This is THE primary cause.

2. **Shaper algorithm bug** in `_AR_FORMS` form selection: the `nextConnectsBackward` flag didn't consider whether THIS letter is itself dual-joining. Right-joining letters (ا د ذ ر ز و) sandwiched between dual-joining letters were incorrectly emitted as medial form (formIdx 3 = final-form variant in our table), looking visually wrong inside words.

## Fix 1: switch to Amiri

Replaced Tajawal with Amiri:

```diff
- /public/fonts/Tajawal-Regular.ttf  (~60KB, incomplete PFB)
- /public/fonts/Tajawal-Bold.ttf     (~60KB)
+ /public/fonts/Amiri-Regular.ttf    (~430KB, FULL PFB coverage)
+ /public/fonts/Amiri-Bold.ttf       (~410KB)
```

Amiri is designed specifically as a complete Arabic typeface (calligraphic Naskh style). It includes EVERY codepoint in Arabic Presentation Forms-B. No silent glyph drops.

The font family is still aliased as "Cairo" in jsPDF's registry — all `pdf.setFont("Cairo")` callers continue to work without modification.

### Trade-off

Bundle size went from ~120KB (Tajawal Regular + Bold) to ~840KB (Amiri Regular + Bold). For an app deploy this is negligible, served once and cached.

The visual look changes: Amiri is calligraphic / traditional Naskh, not modern sans-serif like Cairo or Tajawal. It will look "more like a printed book" than "like a UI app". The user can decide if that's acceptable; if not, we can search for another modern sans Arabic with full PFB coverage in a follow-up.

## Fix 2: shaper algorithm

```diff
- const nextConnectsBackward = _connectsFromPrev(next);
+ const iJoinsForward = entry[4] === true;
+ const nextConnectsBackward = iJoinsForward && _connectsFromPrev(next);
```

The corrected algorithm:
- A letter can connect to next iff (next is connectable) AND (this letter is dual-joining).
- A letter receives connection from prev iff prev is dual-joining (and this letter is in the table, i.e. is Arabic).

### Trace examples

**`الإجمالي`** = ا ل إ ج م ا ل ي

- ا (pos 0): right-joining (iJoinsForward=false), prev=none, next=ل dual.
  - prevConnectsForward = false; nextConnectsBackward = false (because iJoinsForward=false).
  - Form = isolated (0). Glyph = 0xFE8D ﺍ ✓
- ل (pos 1): dual, prev=ا right-joining.
  - prevConnectsForward = false (ا doesn't joinForward); nextConnectsBackward = true (إ is connectable AND ل is dual).
  - Form = initial (2). Glyph = 0xFEDF ﻟ ✓
- إ (pos 2): right-joining, prev=ل dual.
  - prevConnectsForward = true; nextConnectsBackward = false.
  - Form = final (1). Glyph = 0xFE88 ﺈ ✓
- ج (pos 3): dual, prev=إ right-joining.
  - prevConnectsForward = false; nextConnectsBackward = true.
  - Form = initial. Glyph = 0xFE9F ﺟ ✓
- م (pos 4): dual, prev=ج dual.
  - prevConnectsForward = true; nextConnectsBackward = true.
  - Form = medial. Glyph = 0xFEE4 ﻤ ✓
- ا (pos 5): right-joining, prev=م dual.
  - prevConnectsForward = true; nextConnectsBackward = false.
  - Form = final. Glyph = 0xFE8E ﺎ ✓
- ل (pos 6): dual, prev=ا right-joining.
  - prevConnectsForward = false; nextConnectsBackward = true.
  - Form = initial. Glyph = 0xFEDF ﻟ ✓
- ي (pos 7): dual, prev=ل dual.
  - prevConnectsForward = true; nextConnectsBackward = false (no next).
  - Form = final. Glyph = 0xFEF2 ﻲ ✓

Output (logical order): ﺍ ﻟ ﺈ ﺟ ﻤ ﺎ ﻟ ﻲ. Reversed (visual order): ﻲ ﻟ ﺎ ﻤ ﺟ ﺈ ﻟ ﺍ → reads right-to-left as "الإجمالي". Perfect.

## Fix 3: QC-2 label larger

User feedback: "عاوز مربع الـQC يكبر شوية بالارتفاع يكون متناسق، والكلمة تكبر شوية عشان مش واضحة للعين".

```diff
// Box height (relative to QR size)
- const qcStampMM = Math.max(3, Math.round(qrMM * 0.18));
+ const qcStampMM = Math.max(4, Math.round(qrMM * 0.26));

// Inner styling
- border:1.5px solid #000;
- padding:0.4mm Xmm;
- font-size:0.55 of box;
- letter-spacing:1px;
+ border:1.8px solid #000;
+ padding:0.6mm Xmm;     /* X scales 0.6 → 0.7 of box height */
+ font-size:0.7 of box;
+ letter-spacing:1.5px;
+ border-radius:1.2mm;
```

Net effect: ~45% larger box height + ~27% larger text → much more legible at thermal-printer 203/300dpi resolution.
