## التغييرات

### 1. Fix: double-currency suffix

**File:** `src/utils/automation/eventBuilder.js` + `api/_eventBuilder.js`

**قبل:**
```js
if (key === "value" || key === "amount" || key === "balance") return _money(v);
// _money(v) = "12,500 ج.م"
// Template: "القيمة: {amount} ج.م"
// Result:  "القيمة: 12,500 ج.م ج.م"   ← BUG
```

**بعد:**
```js
if (key === "value" || key === "amount" || key === "balance"
 || key === "qty"   || key === "daysLate" || key === "daysToDue") return _fmt(v);
// _fmt(v) = "12,500"
// Template: "القيمة: {amount} ج.م"
// Result:  "القيمة: 12,500 ج.م"   ← CORRECT
```

الـtemplates ما اتغيرتش — هي أصلاً كانت بتـwrite "ج.م" في النص. بس الـauto-formatter ما عادش بـadd duplicate.

### 2. Check Due — only-in-factory filter

**File:** `api/automation-tick.js` (`scanChecksDue`)

**قبل:**
```js
if (c.status === "محصل" || c.status === "مرتد" || c.status === "ملغي") continue;
```

**بعد:**
```js
/* ONLY pending checks (still in our hands).
   Exclude: محصل (cashed), مدفوع (paid out), مُظهّر (endorsed),
            مرتد (bounced), ملغي (cancelled), مرتجع (returned). */
if (c.status !== "معلق") continue;
```

الفرق: الشيك المظهَّر (`مُظهّر`) كان بـraise alert رغم إنه اتنقل لمورد ومش في حوزة المصنع. دلوقتي بس الـ"معلق" (الموجود فعلاً) يـtrigger.

### 3. Check Due — enriched payload

**File:** `api/automation-tick.js` (`scanChecksDue`)

ضافت الـvariables التالية للـpayload:
- `{checkType}` — "ورقة قبض (من عميل)" أو "ورقة دفع (للمورد)"
- `{partyKind}` — "العميل" أو "المورد"
- `{partyName}` — لقيت من الـ`partyId` lookup في customers/suppliers، fallback لـ`c.party`
- `{office}` — الـ`companyName`/`company`/`office`/`businessName` من الـcustomer/supplier record
- `{notes}` — الـ`c.notes`
- `{category}` — الـ`c.category`

**Default template الجديد:**
```
📅 *{checkType} يستحق قريباً*

👤 {partyKind}: {partyName}
🏢 المكتب: {office}
🏦 البنك: {bank}
#️⃣ رقم الشيك: {checkNo}
💰 القيمة: {amount} ج.م
📆 تاريخ الاستحقاق: {dueDate}
⏱ بعد {daysToDue} يوم
📝 {notes}
```

الـ`EVENT_VARIABLES.checkDue.variables.owner` اتحدّثت لتعرض كل الـvariables الجديدة في الـtemplate editor hint.

**ملحوظة:** الـusers الموجودين عندهم الـold template في الـFirestore data. عشان يـpick up الـnew default، اضغط زر "↺ default" في الـtemplate editor. أو إعدّل بإيدك.

### 4. Payment method picker

**File:** `src/pages/TreasuryPg.jsx`

**State:**
```js
const [txMethod, setTxMethod] = useState("نقدي كاش");
```

**UI dropdown** (يظهر فقط لـدفعة عميل/مورد):
```jsx
{((txType==="in"&&txCategory==="دفعة عميل")||(txType==="out"&&txCategory==="دفعة مورد")) &&
  <Sel value={txMethod} onChange={setTxMethod}>
    <option value="نقدي كاش">💵 نقدي كاش</option>
    <option value="تحويل محفظة الكترونية">📱 تحويل محفظة الكترونية</option>
    <option value="تحويل انستاباي">🔄 تحويل انستاباي</option>
    <option value="تحويل بنكي">🏦 تحويل بنكي</option>
  </Sel>}
```

**Save:**
```js
d.custPayments.push({..., method: txMethod || "نقدي كاش", ...});
d.supplierPayments.push({..., method: txMethod || "نقدي كاش", ...});
```

**Edit-time sync:** لما تـedit حركة موجودة:
```js
const linkedPay = (t.custId
  ? data.custPayments.find(p => p.treasuryTxId === t.id)
  : t.supplierId
    ? data.supplierPayments.find(p => p.treasuryTxId === t.id)
    : null);
setTxMethod(linkedPay?.method || "نقدي كاش");
```

الـmethod ده بـappear في الـpaymentReceived event message عبر الـ`{method}` variable.

## How users should adopt the new templates

الـnew enriched template موجود في الـDEFAULT_AUTOMATION_CONFIG لكن الـexisting users عندهم الـold template مسجّل في الـFirestore data. لـpick up الـnew default:

1. Automation page → tab "🔥 Triggers الفورية"
2. افتح الـcheckDue card
3. في الـtemplate editor (لـowner)، اضغط زر "↺ default"
4. الـtemplate هيـreset للـnew enriched version

أو يـdate الـtemplate يدوياً ويستخدم الـnew variables المتاحة في الـvariables hint.
