## ✅ ما تم إنجازه

### 1. Sub-tab "👥 كشوف الموردين" — مفعّل بالكامل

#### 4 Stats Cards:
- **إجمالي المشتريات** (مجموع فواتير كل الموردين)
- **إجمالي المدفوع** (مجموع المدفوعات)
- **الرصيد المستحق** (الإجمالي - المدفوع)
- **موردين لهم رصيد** (عدد الموردين اللي عليهم فلوس)

#### جدول الموردين:
- الاسم + التليفون
- عدد الفواتير
- إجمالي المشتريات
- المدفوع (بالأخضر)
- **الرصيد** ملون:
  - 🔴 أحمر = **عليه** (يعني إحنا ندينله)
  - 🔵 أزرق = **له** (دفعنا أكتر من اللازم — نادر)
  - ⚪ رمادي = **مسدد**
- آخر نشاط
- زر "💰 دفعة" (يظهر فقط لو فيه رصيد عليه)

#### فلاتر متقدمة:
- بحث (الاسم/التليفون) debounced
- ترتيب: الرصيد / الاسم / إجمالي المشتريات / الأحدث

### 2. Statement Popup (كشف الحساب التفصيلي)

عند الضغط على مورد، يفتح popup بـ:

#### Header:
- اسم المورد + التليفون + العنوان
- أزرار: "💰 تسجيل دفعة" + "🖨️ طباعة" + "✕"

#### 4 Summary Cards:
- الفواتير (عدد)
- إجمالي المشتريات
- المدفوع
- الرصيد (ملون حسب الحالة + تسمية واضحة)

#### جدول Journal (محاسبي):
- التاريخ
- النوع (🧾 فاتورة / 💰 دفعة) بـ badges ملونة
- المرجع (رقم الفاتورة أو رقم الشيك)
- البيان
- **مدين** (الفواتير)
- **دائن** (المدفوعات)
- **الرصيد الجاري** ملون
- زر حذف للدفعات (admin فقط)

**الترتيب:** حسب التاريخ ثم النوع (فاتورة قبل دفعة لنفس التاريخ).

**الـ running balance** بيتحسب تلقائياً:
```
balance = Σ(debit) - Σ(credit)
```

### 3. Payment Form Popup

**المدخلات:**
- المبلغ (default = الرصيد الحالي)
- التاريخ
- طريقة الدفع: 💵 كاش / 📄 شيك
- **لو كاش:** اختيار الخزنة
- **لو شيك:** البنك + رقم الشيك + تاريخ الاستحقاق
- ملاحظات

**Validation:**
- المبلغ > 0
- لو كاش: الخزنة مطلوبة
- لو شيك: البنك + الرقم مطلوبين

### 4. Save Payment — Transaction Logic

عند حفظ دفعة، في transaction ذرية:

**دائماً:**
- إضافة في `supplierPayments`

**لو كاش:**
- ✅ إضافة في `treasury` (outflow، category: "دفع مورد")
- ✅ ربط بـ `treasuryTxId`

**لو شيك:**
- ✅ إضافة في `checks` (type: "payable", status: "معلق")
- ✅ ربط بـ `checkId`
- ✅ تظهر في صفحة الشيكات مع باقي المدفوعات المعلقة

### 5. Delete Payment (Rollback Admin-only)

- ✅ حذف من `supplierPayments`
- ✅ حذف من `treasury` (لو كان cash)
- ✅ حذف من `checks` (لو كان معلق)
- ❌ لو الشيك مصروف → لا يُحذف (الفلوس فعلاً خرجت)

### 6. طباعة كشف حساب احترافية

قالب HTML يحتوي على:
- **ترويسة:** CLARK + تاريخ الطباعة
- **بيانات المورد:** اسم، تليفون، عنوان
- **ملخص الحساب:** 3 خانات رئيسية (مشتريات، مدفوع، رصيد)
- **جدول Journal كامل:** كل الحركات بترتيب زمني + running balance
- **ألوان ذكية:** أحمر للرصيد الموجب، أخضر للمسدد، أزرق للسالب
- **Footer:** اسم المورد وتاريخ الطباعة
- **طباعة تلقائية** (بعد 500ms)

## 🔒 الأمان والتكامل

### ✅ Dual source of payments
المدفوعات ممكن تيجي من:
1. مع الاستلام (inline payment في receipt) → يُحسب في `r.paidAmount`
2. standalone payment (من popup الدفع) → يُحسب في `supplierPayments`

**الـ `supplierStats` logic يتجنب double-counting:**
- لو `p.receiptId` موجود → الدفعة اتحسبت في الـ receipt
- لو مش موجود → تُضاف للإجمالي

### ✅ Running balance consistent
ترتيب الـ sort key: `date-type-createdAt`
- نفس التاريخ: فاتورة أول، دفعة بعدها
- يضمن أن الرصيد بعد فاتورة يظهر قبل الدفعة المرتبطة بها

### ✅ Treasury + Checks integration
- كاش → يظهر في الخزنة كـ "دفع مورد"
- شيك → يظهر في الشيكات كـ payable معلق
- الـ `sourceType` + `paymentId` يربطوا الكل

### ✅ Admin-only delete
- الحذف يرجع كل التأثيرات
- الشيك **المصروف** لا يُحذف (safeguard)
